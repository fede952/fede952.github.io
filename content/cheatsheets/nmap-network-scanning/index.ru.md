## $ System_Init

Nmap — это первый инструмент, загружаемый при любой разведывательной деятельности. Он картирует поверхность атаки, идентифицирует активные хосты, перечисляет открытые порты, идентифицирует службы и обнаруживает уязвимости — всё из одного бинарного файла. Это полевое руководство предоставляет точные команды для каждой фазы сетевой разведки.

Все команды предполагают авторизованное тестирование. Используйте ответственно.

---

## $ Host_Discovery

Идентифицировать активные цели в сети перед сканированием портов.

### Ping-развёртка (ICMP echo)

```bash
# Обнаружить активные хосты в подсети с помощью ICMP ping
nmap -sn 192.168.1.0/24
```

### ARP-обнаружение (только локальная сеть)

```bash
# Использовать ARP-запросы для обнаружения хостов в локальной сети (самый быстрый метод)
nmap -sn -PR 192.168.1.0/24
```

### TCP SYN-обнаружение на определённых портах

```bash
# Обнаружить хосты отправкой SYN-пакетов на распространённые порты
nmap -sn -PS22,80,443 10.0.0.0/24
```

### Отключить разрешение DNS (ускорить сканирование)

```bash
# Пропустить обратные DNS-запросы для более быстрых результатов
nmap -sn -n 192.168.1.0/24
```

### Сканирование списка (пакеты не отправляются)

```bash
# Перечислить цели, которые были бы отсканированы, не отправляя пакетов
nmap -sL 192.168.1.0/24
```

## $ Port_Scanning

Перечислить открытые порты для картирования поверхности атаки цели.

### SYN-сканирование (скрытое сканирование — по умолчанию)

```bash
# Полуоткрытое сканирование: отправляет SYN, получает SYN/ACK, отправляет RST (никогда не завершает рукопожатие)
sudo nmap -sS 192.168.1.100
```

### TCP connect сканирование (не требует root)

```bash
# Полное TCP-рукопожатие (медленнее, но работает без повышенных привилегий)
nmap -sT 192.168.1.100
```

### UDP-сканирование

```bash
# Сканировать открытые UDP-порты (медленнее из-за поведения протокола)
sudo nmap -sU 192.168.1.100
```

### Сканировать определённые порты

```bash
# Сканировать только определённые порты
nmap -p 22,80,443,8080 192.168.1.100

# Сканировать диапазон портов
nmap -p 1-1024 192.168.1.100

# Сканировать все 65535 портов
nmap -p- 192.168.1.100
```

### Сканирование топовых портов

```bash
# Сканировать 100 наиболее часто открытых портов
nmap --top-ports 100 192.168.1.100
```

### Быстрое сканирование (топ 100 портов)

```bash
# Быстрое сканирование с уменьшенным количеством портов для быстрой оценки
nmap -F 192.168.1.100
```

## $ Service_Detection

Определить, какое программное обеспечение работает на каждом открытом порту.

### Обнаружение версии

```bash
# Зондировать открытые порты для определения имени и версии службы
nmap -sV 192.168.1.100
```

### Агрессивное обнаружение версии

```bash
# Увеличить интенсивность обнаружения (1-9, по умолчанию 7)
nmap -sV --version-intensity 9 192.168.1.100
```

### Снятие отпечатков ОС

```bash
# Определить операционную систему цели с помощью анализа стека TCP/IP
sudo nmap -O 192.168.1.100
```

### Комбинированное обнаружение службы + ОС

```bash
# Полное перечисление служб со снятием отпечатков ОС
sudo nmap -sV -O 192.168.1.100
```

### Агрессивное сканирование (ОС + версия + скрипты + traceroute)

```bash
# Включить все функции обнаружения одним флагом
sudo nmap -A 192.168.1.100
```

## $ NSE_Scripts

Nmap Scripting Engine — автоматизированное обнаружение уязвимостей и перечисление.

### Запустить скрипты по умолчанию

```bash
# Выполнить стандартный набор безопасных информационных скриптов
nmap -sC 192.168.1.100
```

### Запустить определённый скрипт

```bash
# Выполнить один NSE-скрипт по имени
nmap --script=http-title 192.168.1.100
```

### Запустить категории скриптов

```bash
# Запустить все скрипты обнаружения уязвимостей
nmap --script=vuln 192.168.1.100

# Запустить все скрипты обнаружения
nmap --script=discovery 192.168.1.100

# Запустить скрипты брутфорса против служб аутентификации
nmap --script=brute 192.168.1.100
```

### HTTP-перечисление

```bash
# Перечислить каталоги и файлы веб-сервера
nmap --script=http-enum 192.168.1.100

# Обнаружить межсетевые экраны веб-приложений
nmap --script=http-waf-detect 192.168.1.100
```

### SMB-перечисление

```bash
# Перечислить SMB-ресурсы и пользователей (сети Windows)
nmap --script=smb-enum-shares,smb-enum-users 192.168.1.100
```

### SSL/TLS-анализ

```bash
# Проверить детали SSL-сертификата и наборы шифров
nmap --script=ssl-cert,ssl-enum-ciphers -p 443 192.168.1.100
```

## $ Evasion_Techniques

Обойти межсетевые экраны и IDS во время авторизованных тестов на проникновение.

### Фрагментировать пакеты

```bash
# Разделить зондирующие пакеты на более мелкие фрагменты для обхода простых пакетных фильтров
sudo nmap -f 192.168.1.100
```

### Сканирование с ложными целями

```bash
# Сгенерировать поддельные IP-адреса источника для маскировки реального сканера
sudo nmap -D RND:10 192.168.1.100
```

### Подделать исходный порт

```bash
# Использовать доверенный исходный порт для обхода правил межсетевого экрана на основе портов
sudo nmap --source-port 53 192.168.1.100
```

### Контроль времени

```bash
# T0=Paranoid, T1=Sneaky, T2=Polite, T3=Normal, T4=Aggressive, T5=Insane
nmap -T2 192.168.1.100
```

### Idle-сканирование (зомби-сканирование)

```bash
# Использовать сторонний хост-"зомби" для сканирования без раскрытия вашего IP
sudo nmap -sI zombie-host.com 192.168.1.100
```

## $ Output_Formats

Сохранить результаты сканирования для документации и постобработки.

### Обычный вывод

```bash
# Сохранить результаты в человекочитаемом формате
nmap -oN scan_results.txt 192.168.1.100
```

### XML-вывод (для инструментов)

```bash
# Сохранить результаты в формате XML (анализируемом Metasploit и др.)
nmap -oX scan_results.xml 192.168.1.100
```

### Grep-совместимый вывод

```bash
# Сохранить результаты в формате, удобном для grep и скриптов
nmap -oG scan_results.gnmap 192.168.1.100
```

### Все форматы одновременно

```bash
# Сохранить в обычном, XML и grep-совместимом форматах одновременно
nmap -oA full_scan 192.168.1.100
```

## $ Mission_Templates

Цепочки команд для копирования-вставки для общих сценариев.

### Быстрая разведка

```bash
# Быстрая первоначальная оценка цели
nmap -sS -sV -F -T4 --open 192.168.1.100
```

### Полное сканирование портов с обнаружением служб

```bash
# Комплексное сканирование всех портов с обнаружением версий
sudo nmap -sS -sV -p- -T4 -oA full_scan 192.168.1.100
```

### Оценка уязвимостей

```bash
# Обнаружение служб плюс скрипты уязвимостей
sudo nmap -sV --script=vuln -oA vuln_scan 192.168.1.100
```

### Скрытая разведка (минимальный след)

```bash
# Низкопрофильное сканирование для сред с активным мониторингом
sudo nmap -sS -T2 -f --data-length 24 -D RND:5 -oA stealth_scan 192.168.1.100
```
