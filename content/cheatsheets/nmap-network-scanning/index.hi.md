## $ System_Init

Nmap किसी भी टोही गतिविधि में लोड होने वाला पहला टूल है। यह हमले की सतह को मैप करता है, लाइव होस्ट की पहचान करता है, खुले पोर्ट गिनाता है, सेवाओं का फिंगरप्रिंट लेता है, और कमजोरियों का पता लगाता है — सब एक ही बाइनरी से। यह फील्ड मैनुअल नेटवर्क टोही के प्रत्येक चरण के लिए सटीक कमांड प्रदान करता है।

सभी कमांड अधिकृत परीक्षण मानते हैं। जिम्मेदारी से तैनात करें।

## $ Host_Discovery

पोर्ट स्कैनिंग से पहले नेटवर्क पर लाइव लक्ष्यों की पहचान करें।

### Ping स्वीप (ICMP echo)

```bash
# ICMP ping का उपयोग करके सबनेट पर लाइव होस्ट खोजें
nmap -sn 192.168.1.0/24
```

### ARP डिस्कवरी (केवल लोकल नेटवर्क)

```bash
# स्थानीय LAN पर होस्ट डिस्कवरी के लिए ARP अनुरोध का उपयोग करें (सबसे तेज़ तरीका)
nmap -sn -PR 192.168.1.0/24
```

### विशिष्ट पोर्ट्स पर TCP SYN डिस्कवरी

```bash
# सामान्य पोर्ट्स पर SYN पैकेट भेजकर होस्ट खोजें
nmap -sn -PS22,80,443 10.0.0.0/24
```

### DNS रिज़ॉल्यूशन अक्षम करें (स्कैन तेज़ करें)

```bash
# तेज़ परिणामों के लिए रिवर्स DNS लुकअप छोड़ें
nmap -sn -n 192.168.1.0/24
```

### लिस्ट स्कैन (कोई पैकेट नहीं भेजा गया)

```bash
# बिना कोई पैकेट भेजे स्कैन किए जाने वाले लक्ष्यों की सूची बनाएं
nmap -sL 192.168.1.0/24
```

## $ Port_Scanning

लक्ष्य की हमले की सतह को मैप करने के लिए खुले पोर्ट गिनें।

### SYN स्कैन (स्टेल्थ स्कैन — डिफ़ॉल्ट)

```bash
# हाफ-ओपन स्कैन: SYN भेजता है, SYN/ACK प्राप्त करता है, RST भेजता है (हैंडशेक कभी पूरा नहीं करता)
sudo nmap -sS 192.168.1.100
```

### TCP कनेक्ट स्कैन (रूट की आवश्यकता नहीं)

```bash
# पूर्ण TCP हैंडशेक स्कैन (धीमा लेकिन बिना एलिवेटेड प्रिविलेज के काम करता है)
nmap -sT 192.168.1.100
```

### UDP स्कैन

```bash
# खुले UDP पोर्ट के लिए स्कैन करें (प्रोटोकॉल व्यवहार के कारण धीमा)
sudo nmap -sU 192.168.1.100
```

### विशिष्ट पोर्ट स्कैन करें

```bash
# केवल विशिष्ट पोर्ट स्कैन करें
nmap -p 22,80,443,8080 192.168.1.100

# पोर्ट रेंज स्कैन करें
nmap -p 1-1024 192.168.1.100

# सभी 65535 पोर्ट स्कैन करें
nmap -p- 192.168.1.100
```

### टॉप पोर्ट्स स्कैन

```bash
# सबसे आम तौर पर खुले 100 पोर्ट स्कैन करें
nmap --top-ports 100 192.168.1.100
```

### फास्ट स्कैन (टॉप 100 पोर्ट्स)

```bash
# तेज़ मूल्यांकन के लिए कम पोर्ट संख्या के साथ त्वरित स्कैन
nmap -F 192.168.1.100
```

## $ Service_Detection

प्रत्येक खुले पोर्ट पर चल रहे सॉफ़्टवेयर की पहचान करें।

### वर्जन डिटेक्शन

```bash
# सर्विस नाम और वर्जन निर्धारित करने के लिए खुले पोर्ट की जांच करें
nmap -sV 192.168.1.100
```

### आक्रामक वर्जन डिटेक्शन

```bash
# डिटेक्शन तीव्रता बढ़ाएं (1-9, डिफ़ॉल्ट 7)
nmap -sV --version-intensity 9 192.168.1.100
```

### OS फिंगरप्रिंटिंग

```bash
# TCP/IP स्टैक विश्लेषण का उपयोग करके लक्ष्य का ऑपरेटिंग सिस्टम पहचानें
sudo nmap -O 192.168.1.100
```

### संयुक्त सर्विस + OS डिटेक्शन

```bash
# OS फिंगरप्रिंटिंग के साथ पूर्ण सर्विस एन्यूमरेशन
sudo nmap -sV -O 192.168.1.100
```

### आक्रामक स्कैन (OS + वर्जन + स्क्रिप्ट्स + traceroute)

```bash
# एक फ्लैग में सभी डिटेक्शन सुविधाएं सक्षम करें
sudo nmap -A 192.168.1.100
```

## $ NSE_Scripts

Nmap Scripting Engine — स्वचालित वल्नरेबिलिटी डिटेक्शन और एन्यूमरेशन।

### डिफ़ॉल्ट स्क्रिप्ट चलाएं

```bash
# सुरक्षित, जानकारी देने वाली स्क्रिप्ट के डिफ़ॉल्ट सेट को निष्पादित करें
nmap -sC 192.168.1.100
```

### एक विशिष्ट स्क्रिप्ट चलाएं

```bash
# नाम से एक एकल NSE स्क्रिप्ट निष्पादित करें
nmap --script=http-title 192.168.1.100
```

### स्क्रिप्ट श्रेणियां चलाएं

```bash
# सभी वल्नरेबिलिटी डिटेक्शन स्क्रिप्ट चलाएं
nmap --script=vuln 192.168.1.100

# सभी डिस्कवरी स्क्रिप्ट चलाएं
nmap --script=discovery 192.168.1.100

# ऑथेंटिकेशन सेवाओं के खिलाफ ब्रूट-फ़ोर्स स्क्रिप्ट चलाएं
nmap --script=brute 192.168.1.100
```

### HTTP एन्यूमरेशन

```bash
# वेब सर्वर डायरेक्टरी और फ़ाइलों को गिनें
nmap --script=http-enum 192.168.1.100

# वेब एप्लिकेशन फ़ायरवॉल का पता लगाएं
nmap --script=http-waf-detect 192.168.1.100
```

### SMB एन्यूमरेशन

```bash
# SMB शेयर और यूज़र्स गिनें (Windows नेटवर्क)
nmap --script=smb-enum-shares,smb-enum-users 192.168.1.100
```

### SSL/TLS विश्लेषण

```bash
# SSL सर्टिफिकेट विवरण और साइफर सुइट्स जांचें
nmap --script=ssl-cert,ssl-enum-ciphers -p 443 192.168.1.100
```

## $ Evasion_Techniques

अधिकृत पेनिट्रेशन टेस्ट के दौरान फ़ायरवॉल और IDS को बायपास करें।

### पैकेट फ्रैगमेंट करें

```bash
# सरल पैकेट फ़िल्टर को बायपास करने के लिए प्रोब पैकेट को छोटे टुकड़ों में विभाजित करें
sudo nmap -f 192.168.1.100
```

### डिकॉय स्कैन

```bash
# वास्तविक स्कैनर को छिपाने के लिए नकली स्रोत IP उत्पन्न करें
sudo nmap -D RND:10 192.168.1.100
```

### स्रोत पोर्ट नकली बनाएं

```bash
# पोर्ट-आधारित फ़ायरवॉल नियमों को बायपास करने के लिए एक विश्वसनीय स्रोत पोर्ट का उपयोग करें
sudo nmap --source-port 53 192.168.1.100
```

### टाइमिंग नियंत्रण

```bash
# T0=Paranoid, T1=Sneaky, T2=Polite, T3=Normal, T4=Aggressive, T5=Insane
nmap -T2 192.168.1.100
```

### आइडल स्कैन (ज़ॉम्बी स्कैन)

```bash
# अपना IP प्रकट किए बिना स्कैन करने के लिए थर्ड-पार्टी "ज़ॉम्बी" होस्ट का उपयोग करें
sudo nmap -sI zombie-host.com 192.168.1.100
```

## $ Output_Formats

दस्तावेज़ीकरण और पोस्ट-प्रोसेसिंग के लिए स्कैन परिणाम सहेजें।

### सामान्य आउटपुट

```bash
# मानव-पठनीय प्रारूप में परिणाम सहेजें
nmap -oN scan_results.txt 192.168.1.100
```

### XML आउटपुट (टूल के लिए)

```bash
# XML प्रारूप में परिणाम सहेजें (Metasploit आदि द्वारा पार्स करने योग्य)
nmap -oX scan_results.xml 192.168.1.100
```

### Grep-योग्य आउटपुट

```bash
# स्क्रिप्टिंग के लिए grep-अनुकूल प्रारूप में परिणाम सहेजें
nmap -oG scan_results.gnmap 192.168.1.100
```

### एक साथ सभी प्रारूप

```bash
# एक साथ सामान्य, XML, और grep-योग्य प्रारूपों में सहेजें
nmap -oA full_scan 192.168.1.100
```

## $ Mission_Templates

सामान्य एंगेजमेंट परिदृश्यों के लिए कॉपी-पेस्ट कमांड चेन।

### त्वरित टोही

```bash
# लक्ष्य का तेज़ प्रारंभिक मूल्यांकन
nmap -sS -sV -F -T4 --open 192.168.1.100
```

### सर्विस डिटेक्शन के साथ पूर्ण पोर्ट स्कैन

```bash
# वर्जन डिटेक्शन के साथ सभी पोर्ट की व्यापक स्कैन
sudo nmap -sS -sV -p- -T4 -oA full_scan 192.168.1.100
```

### वल्नरेबिलिटी असेसमेंट

```bash
# सर्विस डिटेक्शन प्लस वल्नरेबिलिटी स्क्रिप्ट्स
sudo nmap -sV --script=vuln -oA vuln_scan 192.168.1.100
```

### स्टेल्थ रिकॉन (न्यूनतम फ़ुटप्रिंट)

```bash
# सक्रिय निगरानी वाले वातावरण के लिए लो-प्रोफ़ाइल स्कैन
sudo nmap -sS -T2 -f --data-length 24 -D RND:5 -oA stealth_scan 192.168.1.100
```
