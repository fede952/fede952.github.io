## $ System_Init

Nmap هي الأداة الأولى التي يتم تحميلها في أي نشاط استطلاع. فهي تقوم بتخطيط سطح الهجوم، وتحديد المضيفين النشطين، وتعداد المنافذ المفتوحة، وبصمة الخدمات، واكتشاف الثغرات الأمنية — كل ذلك من ملف ثنائي واحد. يوفر هذا الدليل الميداني الأوامر الدقيقة لكل مرحلة من مراحل استطلاع الشبكة.

تفترض جميع الأوامر اختبارًا مصرحًا به. قم بالنشر بمسؤولية.

## $ Host_Discovery

تحديد الأهداف النشطة على الشبكة قبل فحص المنافذ.

### مسح Ping (صدى ICMP)

```bash
# اكتشاف المضيفين النشطين على شبكة فرعية باستخدام ping ICMP
nmap -sn 192.168.1.0/24
```

### اكتشاف ARP (الشبكة المحلية فقط)

```bash
# استخدام طلبات ARP لاكتشاف المضيفين على الشبكة المحلية (أسرع طريقة)
nmap -sn -PR 192.168.1.0/24
```

### اكتشاف TCP SYN على منافذ محددة

```bash
# اكتشاف المضيفين عن طريق إرسال حزم SYN إلى المنافذ الشائعة
nmap -sn -PS22,80,443 10.0.0.0/24
```

### تعطيل حل DNS (تسريع الفحص)

```bash
# تخطي عمليات البحث العكسي عن DNS للحصول على نتائج أسرع
nmap -sn -n 192.168.1.0/24
```

### فحص القائمة (لا يتم إرسال حزم)

```bash
# إدراج الأهداف التي سيتم فحصها دون إرسال أي حزم
nmap -sL 192.168.1.0/24
```

## $ Port_Scanning

تعداد المنافذ المفتوحة لتخطيط سطح هجوم الهدف.

### فحص SYN (فحص خفي — افتراضي)

```bash
# فحص نصف مفتوح: يرسل SYN، يستقبل SYN/ACK، يرسل RST (لا يكمل المصافحة أبدًا)
sudo nmap -sS 192.168.1.100
```

### فحص اتصال TCP (لا يتطلب root)

```bash
# فحص مصافحة TCP كامل (أبطأ ولكنه يعمل بدون امتيازات مرتفعة)
nmap -sT 192.168.1.100
```

### فحص UDP

```bash
# فحص منافذ UDP المفتوحة (أبطأ بسبب سلوك البروتوكول)
sudo nmap -sU 192.168.1.100
```

### فحص منافذ محددة

```bash
# فحص منافذ محددة فقط
nmap -p 22,80,443,8080 192.168.1.100

# فحص نطاق من المنافذ
nmap -p 1-1024 192.168.1.100

# فحص جميع المنافذ 65535
nmap -p- 192.168.1.100
```

### فحص المنافذ الرئيسية

```bash
# فحص أكثر 100 منفذ مفتوحًا شيوعًا
nmap --top-ports 100 192.168.1.100
```

### فحص سريع (أفضل 100 منفذ)

```bash
# فحص سريع بعدد منافذ مخفض للتقييم السريع
nmap -F 192.168.1.100
```

## $ Service_Detection

تحديد البرامج التي تعمل على كل منفذ مفتوح.

### كشف الإصدار

```bash
# فحص المنافذ المفتوحة لتحديد اسم الخدمة والإصدار
nmap -sV 192.168.1.100
```

### كشف الإصدار العدواني

```bash
# زيادة شدة الكشف (1-9، الافتراضي 7)
nmap -sV --version-intensity 9 192.168.1.100
```

### بصمة نظام التشغيل

```bash
# كشف نظام تشغيل الهدف باستخدام تحليل حزمة TCP/IP
sudo nmap -O 192.168.1.100
```

### كشف الخدمة + نظام التشغيل المدمج

```bash
# تعداد كامل للخدمات مع بصمة نظام التشغيل
sudo nmap -sV -O 192.168.1.100
```

### فحص عدواني (نظام التشغيل + الإصدار + السكربتات + traceroute)

```bash
# تمكين جميع ميزات الكشف في علامة واحدة
sudo nmap -A 192.168.1.100
```

## $ NSE_Scripts

محرك برمجة Nmap — كشف تلقائي للثغرات الأمنية والتعداد.

### تشغيل السكربتات الافتراضية

```bash
# تنفيذ مجموعة السكربتات الآمنة والمعلوماتية الافتراضية
nmap -sC 192.168.1.100
```

### تشغيل سكربت محدد

```bash
# تنفيذ سكربت NSE واحد بالاسم
nmap --script=http-title 192.168.1.100
```

### تشغيل فئات السكربت

```bash
# تشغيل جميع سكربتات كشف الثغرات الأمنية
nmap --script=vuln 192.168.1.100

# تشغيل جميع سكربتات الاكتشاف
nmap --script=discovery 192.168.1.100

# تشغيل سكربتات القوة الغاشمة ضد خدمات المصادقة
nmap --script=brute 192.168.1.100
```

### تعداد HTTP

```bash
# تعداد أدلة وملفات خادم الويب
nmap --script=http-enum 192.168.1.100

# كشف جدران حماية تطبيقات الويب
nmap --script=http-waf-detect 192.168.1.100
```

### تعداد SMB

```bash
# تعداد مشاركات SMB والمستخدمين (شبكات Windows)
nmap --script=smb-enum-shares,smb-enum-users 192.168.1.100
```

### تحليل SSL/TLS

```bash
# فحص تفاصيل شهادة SSL ومجموعات التشفير
nmap --script=ssl-cert,ssl-enum-ciphers -p 443 192.168.1.100
```

## $ Evasion_Techniques

تجاوز جدران الحماية وأنظمة IDS أثناء اختبارات الاختراق المصرح بها.

### تجزئة الحزم

```bash
# تقسيم حزم الفحص إلى أجزاء أصغر لتجاوز فلاتر الحزم البسيطة
sudo nmap -f 192.168.1.100
```

### فحص خداعي

```bash
# إنشاء عناوين IP مصدر مزيفة لإخفاء الفاحص الحقيقي
sudo nmap -D RND:10 192.168.1.100
```

### تزييف منفذ المصدر

```bash
# استخدام منفذ مصدر موثوق لتجاوز قواعد جدار الحماية المستندة إلى المنفذ
sudo nmap --source-port 53 192.168.1.100
```

### التحكم في التوقيت

```bash
# T0=Paranoid, T1=Sneaky, T2=Polite, T3=Normal, T4=Aggressive, T5=Insane
nmap -T2 192.168.1.100
```

### فحص خامل (فحص الزومبي)

```bash
# استخدام مضيف "زومبي" تابع لطرف ثالث للفحص دون الكشف عن IP الخاص بك
sudo nmap -sI zombie-host.com 192.168.1.100
```

## $ Output_Formats

حفظ نتائج الفحص للتوثيق والمعالجة اللاحقة.

### الإخراج العادي

```bash
# حفظ النتائج بتنسيق قابل للقراءة البشرية
nmap -oN scan_results.txt 192.168.1.100
```

### إخراج XML (للأدوات)

```bash
# حفظ النتائج بتنسيق XML (قابل للتحليل بواسطة Metasploit وما إلى ذلك)
nmap -oX scan_results.xml 192.168.1.100
```

### إخراج قابل لـ Grep

```bash
# حفظ النتائج بتنسيق متوافق مع grep للبرمجة النصية
nmap -oG scan_results.gnmap 192.168.1.100
```

### جميع التنسيقات في وقت واحد

```bash
# الحفظ بالتنسيقات العادية و XML والقابلة لـ grep في وقت واحد
nmap -oA full_scan 192.168.1.100
```

## $ Mission_Templates

سلاسل أوامر النسخ واللصق لسيناريوهات المشاركة الشائعة.

### استطلاع سريع

```bash
# تقييم أولي سريع للهدف
nmap -sS -sV -F -T4 --open 192.168.1.100
```

### فحص كامل للمنافذ مع كشف الخدمة

```bash
# فحص شامل لجميع المنافذ مع كشف الإصدار
sudo nmap -sS -sV -p- -T4 -oA full_scan 192.168.1.100
```

### تقييم الثغرات الأمنية

```bash
# كشف الخدمة بالإضافة إلى سكربتات الثغرات الأمنية
sudo nmap -sV --script=vuln -oA vuln_scan 192.168.1.100
```

### استطلاع خفي (بصمة قليلة)

```bash
# فحص منخفض الملف الشخصي للبيئات ذات المراقبة النشطة
sudo nmap -sS -T2 -f --data-length 24 -D RND:5 -oA stealth_scan 192.168.1.100
```
