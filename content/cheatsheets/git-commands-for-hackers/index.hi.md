## $ System_Init

हर ऑपरेशन एक निशान छोड़ता है। हर कमिट एक चेकपॉइंट है। Git केवल संस्करण नियंत्रण नहीं है — यह हर सॉफ़्टवेयर प्रोजेक्ट की फोरेंसिक रीढ़ है। इस फील्ड मैनुअल में वे कमांड हैं जिन्हें आप दैनिक रूप से उपयोग करेंगे और वे जो आपको बचाएंगे जब सब कुछ टूट जाता है।

कमांड मिशन प्रकार के अनुसार व्यवस्थित हैं। सटीकता से निष्पादित करें।

## $ Emergency_Fixes

जब एक डिप्लॉयमेंट गलत हो जाती है और टाइमलाइन को फिर से लिखने की आवश्यकता होती है।

### अंतिम कमिट को पूर्ववत करें (परिवर्तनों को स्टेजिंग में रखें)

```bash
# अंतिम कमिट को पूर्ववत करें लेकिन अपने परिवर्तनों को स्टेजिंग क्षेत्र में रखें
git reset --soft HEAD~1
```

### अंतिम कमिट को पूर्ववत करें (अनस्टेज परिवर्तन)

```bash
# अंतिम कमिट को पूर्ववत करें और परिवर्तनों को कार्य निर्देशिका में वापस ले जाएं
git reset --mixed HEAD~1
```

### परमाणु रीसेट (सभी स्थानीय परिवर्तनों को नष्ट करें)

```bash
# चेतावनी: यह सभी अप्रतिबद्ध कार्य को स्थायी रूप से नष्ट कर देता है
git reset --hard HEAD~1
```

### अंतिम कमिट संदेश को संशोधित करें

```bash
# नया कमिट बनाए बिना अपने अंतिम कमिट संदेश में टाइपो को ठीक करें
git commit --amend -m "सुधारित कमिट संदेश"
```

### हटाई गई ब्रांच को पुनर्प्राप्त करें

```bash
# reflog में खोए हुए कमिट हैश को खोजें
git reflog

# पुनर्प्राप्त हैश से ब्रांच को फिर से बनाएं
git checkout -b recovered-branch abc1234
```

### इतिहास को फिर से लिखे बिना कमिट को उलटें

```bash
# एक नया कमिट बनाएं जो एक विशिष्ट कमिट को पूर्ववत करता है (साझा ब्रांच के लिए सुरक्षित)
git revert <commit-hash>
```

## $ Stealth_Mode

क्रिप्टोग्राफिक हस्ताक्षर और पहचान सत्यापन। साबित करें कि आपके कमिट प्रामाणिक हैं।

### GPG हस्ताक्षर कॉन्फ़िगर करें

```bash
# अपनी उपलब्ध GPG कुंजियों को सूचीबद्ध करें
gpg --list-secret-keys --keyid-format=long

# Git को बताएं कि कौन सी कुंजी का उपयोग करना है
git config --global user.signingkey YOUR_KEY_ID

# सभी कमिट के लिए स्वचालित हस्ताक्षर सक्षम करें
git config --global commit.gpgsign true
```

### एकल कमिट पर हस्ताक्षर करें

```bash
# मैन्युअल रूप से एक विशिष्ट कमिट पर हस्ताक्षर करें
git commit -S -m "signed: verified deployment"
```

### कमिट हस्ताक्षरों को सत्यापित करें

```bash
# अंतिम कमिट पर हस्ताक्षर की जाँच करें
git log --show-signature -1

# पूरे लॉग में हस्ताक्षरों को सत्यापित करें
git log --pretty="format:%h %G? %aN %s"
```

### रिलीज़ के लिए टैग पर हस्ताक्षर करें

```bash
# एक हस्ताक्षरित रिलीज़ टैग बनाएं
git tag -s v1.0.0 -m "Release v1.0.0 - signed"

# एक हस्ताक्षरित टैग को सत्यापित करें
git tag -v v1.0.0
```

## $ Branch_Operations

समानांतर विकास के लिए सामरिक ब्रांच प्रबंधन।

### नई ब्रांच बनाएं और उस पर स्विच करें

```bash
# एक फीचर ब्रांच बनाएं और एक कमांड में उस पर स्विच करें
git checkout -b feature/new-module
```

### सभी ब्रांच सूचीबद्ध करें (स्थानीय और दूरस्थ)

```bash
# दूरस्थ ट्रैकिंग ब्रांच सहित सभी ब्रांच दिखाएं
git branch -a
```

### ब्रांच को सुरक्षित रूप से हटाएं

```bash
# स्थानीय ब्रांच हटाएं (केवल अगर पूरी तरह से मर्ज की गई है)
git branch -d feature/old-module

# स्थानीय ब्रांच को जबरन हटाएं (भले ही अनमर्ज हो)
git branch -D feature/abandoned-experiment
```

### दूरस्थ ब्रांच हटाएं

```bash
# दूरस्थ रिपॉजिटरी से ब्रांच हटाएं
git push origin --delete feature/old-module
```

### main पर रिबेस करें (रैखिक इतिहास)

```bash
# अपनी ब्रांच के कमिट को नवीनतम main के ऊपर फिर से लागू करें
git checkout feature/my-work
git rebase main
```

### इंटरैक्टिव रिबेस (स्क्वैश, पुनर्व्यवस्थित, संपादित)

```bash
# अंतिम 3 कमिट को इंटरैक्टिव रूप से फिर से लिखें
git rebase -i HEAD~3
```

## $ Reconnaissance

निर्णय लेने से पहले रिपॉजिटरी की स्थिति का निरीक्षण करें।

### ग्राफ के साथ कॉम्पैक्ट लॉग देखें

```bash
# ब्रांच ग्राफ विज़ुअलाइज़ेशन के साथ एक-पंक्ति लॉग
git log --oneline --graph --all --decorate
```

### स्टेजिंग क्षेत्र में परिवर्तन दिखाएं

```bash
# अंतिम कमिट के विरुद्ध स्टेज किए गए परिवर्तनों की तुलना करें
git diff --cached
```

### फ़ाइल को ब्लेम करें (हर पंक्ति को किसने बदला यह खोजें)

```bash
# फ़ाइल में प्रत्येक पंक्ति के लिए लेखक और कमिट दिखाएं
git blame path/to/file.py
```

### कमिट संदेशों में खोजें

```bash
# संदेश में विशिष्ट कीवर्ड वाले कमिट खोजें
git log --grep="bugfix" --oneline
```

### कौन सा कमिट बग लाया यह खोजें

```bash
# कोड तोड़ने वाले परिवर्तन को खोजने के लिए कमिट के माध्यम से बाइनरी खोज
git bisect start
git bisect bad          # वर्तमान कमिट टूटा हुआ है
git bisect good abc1234 # यह पुराना कमिट काम कर रहा था
# Git आपके परीक्षण के लिए कमिट चेकआउट करेगा
```

## $ Stash_Operations

कमिट किए बिना अस्थायी रूप से कार्य को अलग रखें।

### वर्तमान परिवर्तनों को स्टैश करें

```bash
# अप्रतिबद्ध परिवर्तनों को अस्थायी स्टैक में सहेजें
git stash push -m "work in progress: auth module"
```

### सभी स्टैश सूचीबद्ध करें

```bash
# सभी स्टैश की गई प्रविष्टियाँ देखें
git stash list
```

### स्टैश लागू करें और हटाएं

```bash
# सबसे हालिया स्टैश को पुनर्स्थापित करें और स्टैक से हटाएं
git stash pop

# इंडेक्स द्वारा विशिष्ट स्टैश को पुनर्स्थापित करें
git stash apply stash@{2}
```

### स्टैश से ब्रांच बनाएं

```bash
# स्टैश को उचित फीचर ब्रांच में बदलें
git stash branch feature/from-stash stash@{0}
```

## $ Advanced_Protocols

जटिल परिदृश्यों के लिए शक्तिशाली कमांड।

### दूसरी ब्रांच से कमिट चेरी-पिक करें

```bash
# एक ब्रांच से विशिष्ट कमिट को अपनी वर्तमान ब्रांच पर लागू करें
git cherry-pick <commit-hash>
```

### अनट्रैक की गई फ़ाइलों को साफ करें

```bash
# क्या हटाया जाएगा इसका पूर्वावलोकन करें
git clean -n

# अनट्रैक की गई फ़ाइलें और निर्देशिकाएं हटाएं
git clean -fd
```

### पैच फ़ाइल बनाएं

```bash
# अंतिम कमिट को पोर्टेबल पैच फ़ाइल के रूप में निर्यात करें
git format-patch -1 HEAD

# पैच फ़ाइल लागू करें
git am < patch-file.patch
```

### उथला क्लोन (बैंडविड्थ बचाएं)

```bash
# केवल नवीनतम कमिट क्लोन करें (कोई पूर्ण इतिहास नहीं)
git clone --depth 1 https://github.com/user/repo.git
```
