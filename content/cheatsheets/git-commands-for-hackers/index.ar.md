## $ System_Init

كل عملية تترك أثرًا. كل تثبيت هو نقطة تفتيش. Git ليس مجرد نظام للتحكم في الإصدارات — إنه العمود الفقري الجنائي لكل مشروع برمجي. يحتوي هذا الدليل الميداني على الأوامر التي ستستخدمها يوميًا والأوامر التي ستنقذك عندما ينهار كل شيء.

الأوامر منظمة حسب نوع المهمة. نفذ بدقة.

## $ Emergency_Fixes

عندما يفشل النشر وتحتاج إلى إعادة كتابة الجدول الزمني.

### التراجع عن آخر تثبيت (الحفاظ على التغييرات في التدريج)

```bash
# التراجع عن آخر تثبيت لكن الاحتفاظ بالتغييرات في منطقة التدريج
git reset --soft HEAD~1
```

### التراجع عن آخر تثبيت (إزالة التدريج)

```bash
# التراجع عن آخر تثبيت ونقل التغييرات إلى دليل العمل
git reset --mixed HEAD~1
```

### إعادة تعيين نووية (تدمير جميع التغييرات المحلية)

```bash
# تحذير: هذا يدمر جميع الأعمال غير المثبتة بشكل دائم
git reset --hard HEAD~1
```

### تعديل رسالة آخر تثبيت

```bash
# إصلاح خطأ إملائي في رسالة التثبيت الأخيرة دون إنشاء تثبيت جديد
git commit --amend -m "رسالة تثبيت مصححة"
```

### استرداد فرع محذوف

```bash
# العثور على تجزئة التثبيت المفقودة في reflog
git reflog

# إعادة إنشاء الفرع من التجزئة المستردة
git checkout -b recovered-branch abc1234
```

### عكس تثبيت دون إعادة كتابة السجل

```bash
# إنشاء تثبيت جديد يلغي تثبيتًا محددًا (آمن للفروع المشتركة)
git revert <commit-hash>
```

## $ Stealth_Mode

التوقيع التشفيري والتحقق من الهوية. أثبت أن تثبيتاتك أصلية.

### تكوين توقيع GPG

```bash
# عرض قائمة مفاتيح GPG المتاحة
gpg --list-secret-keys --keyid-format=long

# إخبار Git بالمفتاح الذي يجب استخدامه
git config --global user.signingkey YOUR_KEY_ID

# تمكين التوقيع التلقائي لجميع التثبيتات
git config --global commit.gpgsign true
```

### توقيع تثبيت واحد

```bash
# توقيع تثبيت محدد يدويًا
git commit -S -m "signed: verified deployment"
```

### التحقق من توقيعات التثبيت

```bash
# التحقق من التوقيع على آخر تثبيت
git log --show-signature -1

# التحقق من التوقيعات في السجل بأكمله
git log --pretty="format:%h %G? %aN %s"
```

### توقيع الوسوم للإصدارات

```bash
# إنشاء وسم إصدار موقع
git tag -s v1.0.0 -m "Release v1.0.0 - signed"

# التحقق من وسم موقع
git tag -v v1.0.0
```

## $ Branch_Operations

إدارة تكتيكية للفروع للتطوير المتوازي.

### إنشاء والانتقال إلى فرع جديد

```bash
# إنشاء فرع ميزة والانتقال إليه في أمر واحد
git checkout -b feature/new-module
```

### عرض قائمة بجميع الفروع (المحلية والبعيدة)

```bash
# عرض جميع الفروع بما في ذلك فروع التتبع البعيدة
git branch -a
```

### حذف فرع بأمان

```bash
# حذف فرع محلي (فقط إذا كان مدمجًا بالكامل)
git branch -d feature/old-module

# فرض حذف فرع محلي (حتى إذا لم يكن مدمجًا)
git branch -D feature/abandoned-experiment
```

### حذف فرع بعيد

```bash
# إزالة فرع من المستودع البعيد
git push origin --delete feature/old-module
```

### إعادة الأساس على main (سجل خطي)

```bash
# إعادة تطبيق تثبيتات فرعك على أحدث main
git checkout feature/my-work
git rebase main
```

### إعادة أساس تفاعلية (ضغط، إعادة ترتيب، تحرير)

```bash
# إعادة كتابة آخر 3 تثبيتات بشكل تفاعلي
git rebase -i HEAD~3
```

## $ Reconnaissance

فحص حالة المستودع قبل اتخاذ القرارات.

### عرض سجل مضغوط مع رسم بياني

```bash
# سجل سطر واحد مع تصور رسومي للفروع
git log --oneline --graph --all --decorate
```

### عرض التغييرات في منطقة التدريج

```bash
# مقارنة التغييرات المرحلية مع آخر تثبيت
git diff --cached
```

### إلقاء اللوم على ملف (العثور على من غيّر كل سطر)

```bash
# عرض المؤلف والتثبيت لكل سطر في ملف
git blame path/to/file.py
```

### البحث في رسائل التثبيت

```bash
# العثور على تثبيتات تحتوي على كلمة رئيسية محددة في الرسالة
git log --grep="bugfix" --oneline
```

### العثور على التثبيت الذي أدخل خطأ

```bash
# بحث ثنائي خلال التثبيتات للعثور على التغيير الذي كسر الكود
git bisect start
git bisect bad          # التثبيت الحالي معطل
git bisect good abc1234 # هذا التثبيت القديم كان يعمل
# سيقوم Git بسحب التثبيتات لك لاختبارها
```

## $ Stash_Operations

تخزين العمل مؤقتًا دون تثبيت.

### تخزين التغييرات الحالية

```bash
# حفظ التغييرات غير المثبتة في مكدس مؤقت
git stash push -m "work in progress: auth module"
```

### عرض قائمة بجميع التخزينات

```bash
# عرض جميع إدخالات التخزين
git stash list
```

### تطبيق وإزالة تخزين

```bash
# استعادة أحدث تخزين وإزالته من المكدس
git stash pop

# استعادة تخزين محدد بالفهرس
git stash apply stash@{2}
```

### إنشاء فرع من تخزين

```bash
# تحويل تخزين إلى فرع ميزة مناسب
git stash branch feature/from-stash stash@{0}
```

## $ Advanced_Protocols

أوامر قوية للسيناريوهات المعقدة.

### اختيار تثبيت من فرع آخر

```bash
# تطبيق تثبيت محدد من فرع إلى فرعك الحالي
git cherry-pick <commit-hash>
```

### تنظيف الملفات غير المتعقبة

```bash
# معاينة ما سيتم حذفه
git clean -n

# حذف الملفات والأدلة غير المتعقبة
git clean -fd
```

### إنشاء ملف رقعة

```bash
# تصدير آخر تثبيت كملف رقعة محمول
git format-patch -1 HEAD

# تطبيق ملف رقعة
git am < patch-file.patch
```

### استنساخ سطحي (توفير النطاق الترددي)

```bash
# استنساخ آخر تثبيت فقط (بدون سجل كامل)
git clone --depth 1 https://github.com/user/repo.git
```
