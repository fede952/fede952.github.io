## Зачем сканировать до коммита

Достаточно одного `.env` в публичном репозитории: боты сканируют GitHub и находят свежие ключи AWS **менее чем за минуту**. SafeEnv ловит утечку до коммита. Вставьте любую конфигурацию — `.env`, `docker-compose.yml`, CI-конфиг, фрагменты кода — и он отметит открытые учётные данные с номером строки, замаскированным превью и конкретными шагами по устранению.

Сканирование выполняется полностью в памяти этой страницы. Без загрузки, без логов, без сетевых запросов — единственный приемлемый дизайн для инструмента, куда вставляют настоящие секреты. Перезагрузите страницу — и всё исчезнет.

## Что обнаруживает

- **Облачные и API-токены** — ключи AWS, GitHub, GitLab, Stripe, Google, OpenAI, Anthropic, Slack, SendGrid, npm, PyPI, Telegram, Twilio
- **Приватные ключи** — PEM-блоки RSA/EC/OpenSSH/PGP
- **Учётные данные в URL** — строки подключения к базам данных и basic-auth URL со встроенными паролями
- **Общие утечки** — захардкоженные пароли и значения с высокой энтропией, с распознаванием заглушек для снижения ложных срабатываний

Вставьте конфигурацию для проверки или загрузите пример, чтобы увидеть все детекторы в действии на фейковых ключах.
