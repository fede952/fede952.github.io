## $ System_Init

석양의 사진. 프로필 사진. 소셜 미디어에서 공유된 밈. 모든 관찰자에게 그것들은 평범한 이미지 파일입니다. 그러나 픽셀 데이터 내부에 묻혀 있는 — 인간의 눈에는 보이지 않는 — 어디를 봐야 할지 아는 사람에 의해 추출되기를 기다리는 숨겨진 메시지가 있을 수 있습니다.

이것이 **스테가노그래피**입니다: 정보를 눈에 띄는 곳에 숨기는 기술. 데이터를 읽을 수 없는 암호문으로 변환하는 암호화(따라서 비밀이 존재한다는 것을 알림)와 달리, 스테가노그래피는 비밀의 존재 자체를 숨깁니다. 파일을 스캔하는 적은 아무것도 비정상적인 것을 보지 못합니다 — 단지 또 다른 JPEG, 단지 또 다른 PNG일 뿐입니다.

이 가이드는 가장 일반적인 디지털 스테가노그래피 기술인 **최하위 비트(LSB) 삽입**을 첫 번째 원리부터 설명합니다. 끝까지 읽으면 정확히 어떻게 작동하는지, 왜 거의 감지할 수 없는지, 그리고 한계가 어디에 있는지 이해하게 될 것입니다.

## $ What_Is_Steganography

이 단어는 그리스어에서 유래했습니다: *steganos*(덮인) + *graphein*(쓰기). 문자 그대로 "덮인 글".

스테가노그래피는 수천 년 동안 존재해 왔습니다. 헤로도토스는 머리를 밀고, 두개골에 비밀 메시지를 문신하고, 머리카락이 다시 자라기를 기다린 다음 적 영토를 여행한 그리스 전령들을 묘사했습니다. 전령의 머리를 밀어야 한다는 것을 알지 못하면 메시지는 보이지 않았습니다.

디지털 시대에서 원리는 동일하지만 매체가 변경되었습니다. 인간 피부 대신 **이미지 파일**을 사용합니다. 문신 잉크 대신 **비트 조작**을 사용합니다.

### 스테가노그래피 vs 암호화

| 속성 | 암호화 | 스테가노그래피 |
|---|---|---|
| **목표** | 데이터를 읽을 수 없게 만들기 | 데이터를 보이지 않게 만들기 |
| **가시성** | 암호문은 보임(무언가가 암호화되었음이 명백함) | 캐리어 파일이 정상으로 보임 |
| **탐지** | 탐지하기 쉽고, 해독하기 어려움 | 탐지하기 어렵고, 발견되면 추출하기 쉬움 |
| **최선의 사용** | 데이터 기밀성 보호 | 통신이 일어나고 있다는 사실 숨기기 |

가장 강력한 접근 방식은 둘 다 결합합니다: 먼저 메시지를 암호화한 다음 스테가노그래피를 사용하여 암호문을 포함시킵니다. 숨겨진 데이터가 발견되더라도 복호화 키 없이는 읽을 수 없습니다.

## $ How_LSB_Works

디지털 이미지는 픽셀로 구성됩니다. 각 픽셀은 색상 값을 저장합니다 — 일반적으로 빨강, 녹색, 파랑(RGB) — 각 채널은 8비트(값 0-255)를 사용합니다.

색상 값 `R=148, G=203, B=72`를 가진 단일 픽셀을 생각해 보세요. 이진수로:

```
R: 10010100
G: 11001011
B: 01001000
```

**최하위 비트**는 각 바이트의 가장 오른쪽 비트입니다. 이것을 변경하면 색상 값이 256 중 최대 1만큼 변경됩니다 — **0.39%**의 차이로 인간의 눈에는 완전히 보이지 않습니다.

### 메시지 포함

문자 `H`(ASCII 72, 이진수 `01001000`)를 세 개의 픽셀에 숨기려면:

```
Original pixels (RGB):
Pixel 1: (148, 203, 72)  → 10010100  11001011  01001000
Pixel 2: (55, 120, 91)   → 00110111  01111000  01011011
Pixel 3: (200, 33, 167)  → 11001000  00100001  10100111

Message bits: 0 1 0 0 1 0 0 0

After LSB replacement:
Pixel 1: (148, 203, 72)  → 10010100  11001011  01001000
Pixel 2: (54, 121, 90)   → 00110110  01111001  01011010
Pixel 3: (200, 32, 167)  → 11001000  00100000  10100111
```

수정된 픽셀은 단일 채널에서 최대 1만큼 차이가 납니다. 이미지는 동일하게 보입니다.

### 용량

각 픽셀은 3비트를 저장합니다(RGB 채널당 하나). 1920x1080 이미지에는 2,073,600개의 픽셀이 있어 이론적 용량은 다음과 같습니다:

```
2,073,600 pixels × 3 bits = 6,220,800 bits = 777,600 bytes ≈ 759 KB
```

이것은 단일 사진 내부에 전체 문서를 숨기기에 충분합니다.

## $ Detection_And_Limits

LSB 스테가노그래피는 완벽하지 않습니다. 알려진 취약점은 다음과 같습니다:

### 통계적 분석(스테가널리시스)

깨끗한 이미지는 픽셀 값에 자연스러운 통계적 패턴을 가지고 있습니다. LSB 삽입은 이러한 패턴을 방해합니다. **StegExpose** 및 **카이제곱 분석**과 같은 도구는 비트 교체로 인해 도입된 통계적 이상을 감지할 수 있습니다 — 특히 메시지가 캐리어 이미지에 비해 클 때.

### 압축은 페이로드를 파괴합니다

JPEG 압축은 **손실**입니다 — 인코딩 중에 픽셀 값을 수정합니다. 이것은 LSB 데이터를 파괴합니다. 스테가노그래픽 페이로드는 PNG, BMP 또는 TIFF와 같은 **무손실 형식**에서만 살아남습니다. PNG에 메시지를 포함시킨 다음 JPEG로 변환하면 메시지가 사라집니다.

### 이미지 조작은 페이로드를 파괴합니다

크기 조정, 자르기, 회전 또는 필터 적용(밝기, 대비 등)은 모두 픽셀 값을 수정하고 숨겨진 데이터를 파괴합니다. 캐리어 이미지는 수정 없이 전송 및 저장되어야 합니다.

### 모범 사례

- 높은 엔트로피를 가진 **큰 이미지**를 사용(사진, 단색이나 그라데이션 아님)
- **PNG 형식**을 사용(무손실 압축은 페이로드를 보존함)
- 포함하기 전에 **메시지를 암호화**(심층 방어)
- 메시지 크기를 **캐리어 용량의 10% 미만**으로 유지하여 통계적 감지 가능성을 최소화

## $ Try_It_Yourself

이론은 실천 없이는 아무것도 아닙니다. 무료 클라이언트 측 **[스테가노그래피 랩](/tools/steganography/)**을 사용하여 브라우저에서 직접 이미지에 숨겨진 메시지를 인코딩하세요.

업로드 없음, 서버 처리 없음. 데이터는 컴퓨터에 남습니다.

1. [스테가노그래피 랩](/tools/steganography/)을 엽니다
2. 캐리어 이미지를 업로드합니다(PNG 권장)
3. 비밀 메시지를 입력합니다
4. 인코딩을 클릭합니다 — 도구는 LSB 삽입을 사용하여 메시지를 포함시킵니다
5. 출력 이미지를 다운로드합니다
6. 어디를 확인해야 하는지 아는 사람과 공유합니다
7. 그들이 업로드하고 디코딩을 클릭하여 메시지를 읽습니다

## $ FAQ_Database

**스테가노그래피를 탐지할 수 있나요?**

예, 통계적 분석(스테가널리시스)을 통해. 도구는 LSB 삽입이 픽셀 값 분포에 미치는 미묘한 변화를 감지할 수 있습니다. 그러나 탐지에는 적극적인 의심이 필요합니다 — 찾을 이유가 없으면 아무도 임의의 이미지를 숨겨진 데이터로 분석하지 않습니다. 큰 고엔트로피 이미지에 작은 메시지를 사용하면 탐지가 훨씬 더 어려워집니다.

**스테가노그래피는 불법인가요?**

스테가노그래피 자체는 기술이지 범죄가 아닙니다. 대부분의 관할권에서 합법입니다. 그러나 불법 활동(도난 데이터 전송, 아동 착취 자료 등)을 촉진하는 데 사용하는 것은 불법입니다 — 잠긴 금고는 합법이지만 밀수품을 숨기는 것은 불법인 것처럼. 이 도구는 교육 목적과 합법적인 개인 정보 보호 사용 사례를 위해 제공됩니다.

**왜 암호화만 사용하지 않나요?**

암호화는 메시지의 내용을 보호하지만 메시지가 존재한다는 사실은 보호하지 못합니다. 일부 위협 모델(억압적인 체제, 기업 감시, 검열)에서는 암호화된 통신을 보내는 행위 자체가 주목을 받습니다. 스테가노그래피는 통신 자체를 숨깁니다. 이상적인 접근 방식은 먼저 암호화한 다음 포함시키는 것입니다 — 메시지는 보이지 않고 읽을 수 없습니다.

**소셜 미디어가 스테가노그래픽 페이로드를 파괴하나요?**

예. Instagram, Twitter/X, Facebook, WhatsApp과 같은 플랫폼은 업로드된 이미지를 압축하고 크기를 조정하여 LSB 데이터를 파괴합니다. 스테가노그래픽 이미지를 전송하려면 원본 파일을 보존하는 채널을 사용하세요: 이메일 첨부 파일, 클라우드 스토리지 링크 또는 직접 파일 전송.
