## $ System_Init

夕日の写真。プロフィール画像。ソーシャルメディアで共有されたミーム。どの観察者にとっても、それらは通常の画像ファイルです。しかし、ピクセルデータの中に埋もれた — 人間の目には見えない — どこを見ればいいか知っている誰かによって抽出されるのを待っている隠されたメッセージがあるかもしれません。

これが**ステガノグラフィー**です：目に見える場所に情報を隠す技術。暗号化がデータを読めない暗号文にスクランブル化し（したがって秘密が存在することを告知する）のとは異なり、ステガノグラフィーは秘密の存在そのものを隠します。あなたのファイルをスキャンする敵対者は何も異常を見ません — ただの別のJPEG、ただの別のPNG。

このガイドでは、最も一般的なデジタルステガノグラフィー技術 — **最下位ビット（LSB）挿入** — を第一原理から説明します。最後には、それがどのように機能するか、なぜほぼ検出不可能なのか、そしてその限界がどこにあるのかを正確に理解できるでしょう。

## $ What_Is_Steganography

この言葉はギリシャ語に由来します：*steganos*（覆われた）+ *graphein*（書く）。文字通り、「覆われた書き物」。

ステガノグラフィーは何千年も前から存在しています。ヘロドトスは、頭を剃り、頭蓋骨に秘密のメッセージをタトゥーし、髪が生え戻るのを待ち、その後敵の領土を旅したギリシャの使者について述べています。メッセージは、使者の頭を剃ることを知らない限り見えませんでした。

デジタル時代では、原理は同じですが、媒体が変わりました。人間の皮膚の代わりに、**画像ファイル**を使用します。タトゥーのインクの代わりに、**ビット操作**を使用します。

### ステガノグラフィー vs 暗号化

| 特性 | 暗号化 | ステガノグラフィー |
|---|---|---|
| **目標** | データを読めなくする | データを見えなくする |
| **可視性** | 暗号文は可視（何かが暗号化されていることが明らか） | キャリアファイルは正常に見える |
| **検出** | 検出は容易、解読は困難 | 検出は困難、発見されれば抽出は容易 |
| **最良の使用** | データの機密性を保護 | 通信が行われている事実を隠す |

最も強力なアプローチは両方を組み合わせます：まずメッセージを暗号化し、次にステガノグラフィーを使用して暗号文を埋め込みます。隠されたデータが発見されても、復号化キーがなければ読めないままです。

## $ How_LSB_Works

デジタル画像はピクセルで構成されています。各ピクセルは色値を格納します — 通常は赤、緑、青（RGB）— 各チャネルは8ビット（値0-255）を使用します。

色値 `R=148, G=203, B=72` の単一ピクセルを考えてみましょう。バイナリでは：

```
R: 10010100
G: 11001011
B: 01001000
```

**最下位ビット**は各バイトの最も右のビットです。これを変更すると、色値が256のうち最大1変化します — **0.39%**の違いで、人間の目には完全に見えません。

### メッセージの埋め込み

文字 `H`（ASCII 72、バイナリ `01001000`）を3つのピクセルに隠すには：

```
Original pixels (RGB):
Pixel 1: (148, 203, 72)  → 10010100  11001011  01001000
Pixel 2: (55, 120, 91)   → 00110111  01111000  01011011
Pixel 3: (200, 33, 167)  → 11001000  00100001  10100111

Message bits: 0 1 0 0 1 0 0 0

After LSB replacement:
Pixel 1: (148, 203, 72)  → 10010100  11001011  01001000
Pixel 2: (54, 121, 90)   → 00110110  01111001  01011010
Pixel 3: (200, 32, 167)  → 11001000  00100000  10100111
```

変更されたピクセルは単一チャネルで最大1だけ異なります。画像は同一に見えます。

### 容量

各ピクセルは3ビットを格納します（RGBチャネルごとに1つ）。1920x1080の画像には2,073,600ピクセルがあり、理論的容量は次のようになります：

```
2,073,600 pixels × 3 bits = 6,220,800 bits = 777,600 bytes ≈ 759 KB
```

これは、単一の写真内に文書全体を隠すのに十分です。

## $ Detection_And_Limits

LSBステガノグラフィーは完璧ではありません。既知の脆弱性は次のとおりです：

### 統計分析（ステガナリシス）

クリーンな画像には、ピクセル値に自然な統計パターンがあります。LSB挿入はこれらのパターンを乱します。**StegExpose**や**カイ二乗分析**などのツールは、ビット置換によって導入される統計的異常を検出できます — 特にメッセージがキャリア画像に対して大きい場合。

### 圧縮はペイロードを破壊します

JPEG圧縮は**非可逆**です — エンコード中にピクセル値を変更します。これによりLSBデータが破壊されます。ステガノグラフィーペイロードは、PNG、BMP、TIFFなどの**可逆形式**でのみ生き残ります。PNGにメッセージを埋め込んでからJPEGに変換すると、メッセージは失われます。

### 画像操作はペイロードを破壊します

リサイズ、クロップ、回転、またはフィルタの適用（明るさ、コントラストなど）はすべてピクセル値を変更し、隠されたデータを破壊します。キャリア画像は変更せずに送信および保存する必要があります。

### ベストプラクティス

- 高エントロピーの**大きな画像**を使用（写真、単色やグラデーションではない）
- **PNG形式**を使用（可逆圧縮はペイロードを保持）
- 埋め込み前に**メッセージを暗号化**（多層防御）
- メッセージサイズを**キャリア容量の10%未満**に保ち、統計的検出可能性を最小限に抑える

## $ Try_It_Yourself

理論は実践なしでは何もありません。無料のクライアント側**[ステガノグラフィー・ラボ](/tools/steganography/)**を使用して、独自の隠しメッセージを画像にエンコードしましょう — ブラウザで直接。

アップロードなし、サーバー処理なし。データはマシン上に留まります。

1. [ステガノグラフィー・ラボ](/tools/steganography/)を開く
2. キャリア画像をアップロード（PNG推奨）
3. 秘密のメッセージを入力
4. エンコードをクリック — ツールはLSB挿入を使用してメッセージを埋め込みます
5. 出力画像をダウンロード
6. どこを確認すればいいか知っている人と共有
7. 彼らがアップロードし、デコードをクリックして、あなたのメッセージを読む

## $ FAQ_Database

**ステガノグラフィーは検出できますか？**

はい、統計分析（ステガナリシス）によって。ツールは、LSB挿入がピクセル値分布に与える微妙な変化を検出できます。ただし、検出には積極的な疑いが必要です — 探す理由がない限り、誰もランダムな画像を隠しデータで分析しません。大きな高エントロピー画像に小さなメッセージを使用すると、検出が大幅に困難になります。

**ステガノグラフィーは違法ですか？**

ステガノグラフィー自体は技術であり、犯罪ではありません。ほとんどの管轄区域で合法です。ただし、違法活動（盗まれたデータの送信、児童搾取資料など）を促進するために使用することは違法です — ロックされた金庫は合法ですが、密輸品を隠すことは違法であるのと同じです。このツールは、教育目的および正当なプライバシー使用ケースのために提供されています。

**なぜ暗号化だけを使わないのですか？**

暗号化はメッセージの内容を保護しますが、メッセージが存在するという事実は保護しません。一部の脅威モデル（抑圧的な体制、企業監視、検閲）では、暗号化された通信を送信するという単なる行為が注目を集めます。ステガノグラフィーは通信自体を隠します。理想的なアプローチは、まず暗号化し、次に埋め込むことです — メッセージは見えず、読めません。

**ソーシャルメディアはステガノグラフィーペイロードを破壊しますか？**

はい。Instagram、Twitter/X、Facebook、WhatsAppなどのプラットフォームは、アップロードされた画像を圧縮およびリサイズし、LSBデータを破壊します。ステガノグラフィー画像を送信するには、元のファイルを保持するチャネルを使用してください：メール添付、クラウドストレージリンク、または直接ファイル転送。
