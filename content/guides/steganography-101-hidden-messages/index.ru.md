## $ System_Init

Фотография заката. Аватар профиля. Мем, опубликованный в социальных сетях. Для любого наблюдателя это обычные файлы изображений. Но скрытое внутри данных пикселей — невидимое для человеческого глаза — может находиться тайное сообщение, ожидающее извлечения тем, кто знает, где искать.

Это **стеганография**: искусство скрывать информацию на виду. В отличие от шифрования, которое превращает данные в нечитаемый шифротекст (и таким образом объявляет о существовании секрета), стеганография скрывает само существование секрета. Противник, сканирующий ваши файлы, не видит ничего необычного — просто еще один JPEG, просто еще один PNG.

Это руководство объясняет наиболее распространенную технику цифровой стеганографии — **вставку Наименее Значащего Бита (LSB)** — с самых основ. В конце вы поймете, как именно это работает, почему это почти необнаруживаемо и где находятся его пределы.

## $ What_Is_Steganography

Слово происходит от греческого: *steganos* (покрытый) + *graphein* (письмо). Буквально, "скрытое письмо."

Стеганография существует тысячелетиями. Геродот описывал греческих гонцов, которые брили головы, татуировали секретные сообщения на черепах, ждали, пока волосы отрастут, а затем путешествовали по вражеской территории. Сообщение было невидимым, если только вы не знали, что нужно побрить голову гонца.

В цифровую эпоху принцип идентичен — но носитель изменился. Вместо человеческой кожи мы используем **файлы изображений**. Вместо чернил для тату мы используем **манипуляцию битами**.

### Стеганография против Шифрования

| Свойство | Шифрование | Стеганография |
|---|---|---|
| **Цель** | Сделать данные нечитаемыми | Сделать данные невидимыми |
| **Видимость** | Шифротекст виден (очевидно, что что-то зашифровано) | Файл-носитель выглядит нормально |
| **Обнаружение** | Легко обнаружить, трудно взломать | Трудно обнаружить, легко извлечь после обнаружения |
| **Лучшее Использование** | Защитить конфиденциальность данных | Скрыть факт происходящей коммуникации |

Самый мощный подход сочетает оба метода: сначала зашифруйте сообщение, затем встройте шифротекст с помощью стеганографии. Даже если скрытые данные будут обнаружены, они остаются нечитаемыми без ключа дешифрования.

## $ How_LSB_Works

Цифровые изображения состоят из пикселей. Каждый пиксель хранит значения цвета — обычно Красный, Зеленый и Синий (RGB) — причем каждый канал использует 8 бит (значения 0-255).

Рассмотрим один пиксель со значением цвета `R=148, G=203, B=72`. В двоичной системе:

```
R: 10010100
G: 11001011
B: 01001000
```

**Наименее Значащий Бит** — это самый правый бит в каждом байте. Его изменение изменяет значение цвета максимум на 1 из 256 — разница в **0,39%**, которая совершенно невидима для человеческого глаза.

### Встраивание сообщения

Чтобы скрыть букву `H` (ASCII 72, двоичное `01001000`) в трех пикселях:

```
Original pixels (RGB):
Pixel 1: (148, 203, 72)  → 10010100  11001011  01001000
Pixel 2: (55, 120, 91)   → 00110111  01111000  01011011
Pixel 3: (200, 33, 167)  → 11001000  00100001  10100111

Message bits: 0 1 0 0 1 0 0 0

After LSB replacement:
Pixel 1: (148, 203, 72)  → 10010100  11001011  01001000
Pixel 2: (54, 121, 90)   → 00110110  01111001  01011010
Pixel 3: (200, 32, 167)  → 11001000  00100000  10100111
```

Модифицированные пиксели отличаются максимум на 1 в одном канале. Изображение выглядит идентично.

### Емкость

Каждый пиксель хранит 3 бита (один на канал RGB). Изображение 1920x1080 имеет 2 073 600 пикселей, что дает теоретическую емкость:

```
2,073,600 pixels × 3 bits = 6,220,800 bits = 777,600 bytes ≈ 759 KB
```

Этого достаточно, чтобы скрыть целый документ внутри одной фотографии.

## $ Detection_And_Limits

LSB-стеганография не идеальна. Вот известные уязвимости:

### Статистический анализ (Стеганоанализ)

Чистые изображения имеют естественные статистические паттерны в значениях пикселей. Вставка LSB нарушает эти паттерны. Инструменты, такие как **StegExpose** и **анализ хи-квадрат**, могут обнаружить статистические аномалии, вызванные заменой битов — особенно когда сообщение велико по сравнению с изображением-носителем.

### Сжатие уничтожает полезную нагрузку

Сжатие JPEG является **с потерями** — оно изменяет значения пикселей во время кодирования. Это уничтожает данные LSB. Стеганографические полезные нагрузки выживают только в **форматах без потерь**, таких как PNG, BMP или TIFF. Если вы встроите сообщение в PNG, а затем конвертируете его в JPEG, сообщение исчезнет.

### Манипуляция изображением уничтожает полезную нагрузку

Изменение размера, обрезка, поворот или применение фильтров (яркость, контраст и т.д.) изменяют значения пикселей и уничтожают скрытые данные. Изображение-носитель должно передаваться и храниться без изменений.

### Лучшие практики

- Используйте **большие изображения** с высокой энтропией (фотографии, а не сплошные цвета или градиенты)
- Используйте **формат PNG** (сжатие без потерь сохраняет полезную нагрузку)
- **Зашифруйте сообщение** перед встраиванием (эшелонированная защита)
- Держите размер сообщения **ниже 10% от емкости носителя**, чтобы минимизировать статистическую обнаруживаемость

## $ FAQ_Database

**Можно ли обнаружить стеганографию?**

Да, с помощью статистического анализа (стеганоанализа). Инструменты могут обнаружить тонкие изменения, которые вставка LSB вносит в распределения значений пикселей. Однако обнаружение требует активного подозрения — никто не анализирует случайные изображения на наличие скрытых данных, если у них нет причины искать. Использование небольших сообщений в больших изображениях с высокой энтропией значительно затрудняет обнаружение.

**Является ли стеганография незаконной?**

Стеганография сама по себе — это техника, а не преступление. Она легальна в большинстве юрисдикций. Однако использование её для содействия незаконной деятельности (передача украденных данных, материалов эксплуатации детей и т.д.) незаконно — так же как запертый сейф законен, но сокрытие в нём контрабанды — нет. Этот инструмент предоставляется в образовательных целях и для законных случаев использования конфиденциальности.

**Почему бы просто не использовать шифрование?**

Шифрование защищает содержимое сообщения, но не факт существования сообщения. В некоторых моделях угроз (репрессивные режимы, корпоративная слежка, цензура) сам акт отправки зашифрованной коммуникации привлекает внимание. Стеганография скрывает саму коммуникацию. Идеальный подход — сначала зашифровать, затем встроить — сообщение является одновременно невидимым и нечитаемым.

**Уничтожают ли социальные сети стеганографические полезные нагрузки?**

Да. Платформы, такие как Instagram, Twitter/X, Facebook и WhatsApp, сжимают и изменяют размер загруженных изображений, что уничтожает данные LSB. Для передачи стеганографических изображений используйте каналы, которые сохраняют исходный файл: вложения электронной почты, ссылки на облачное хранилище или прямую передачу файлов.
