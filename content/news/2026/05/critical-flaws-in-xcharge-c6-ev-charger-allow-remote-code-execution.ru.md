CISA предупреждает о неаутентифицированных уязвимостях в контроллерах зарядных устройств XCharge C6, включая CVE-2026-9037 с оценкой CVSS 9.8.

{{< cyber-report severity="Critical" source="CISA" target="Контроллеры зарядных устройств XCharge C6" cve="CVE-2026-9037" cvss="9.8" >}}

CISA выпустила предупреждение (ICSA-26-148-08), в котором описываются множественные критические уязвимости в контроллерах зарядных устройств для электромобилей XCharge C6. Недостатки включают загрузку кода без проверки целостности (CWE-494), переполнение буфера на основе стека и инициализацию ресурса с небезопасными настройками по умолчанию. Успешная эксплуатация может позволить злоумышленнику получить права администратора или выполнить произвольный код на устройстве.

{{< ad-banner >}}

Наиболее серьезная уязвимость, CVE-2026-9037, связана с механизмом обновления прошивки, который не проверяет подлинность пакетов прошивки. Без проверки криптографической подписи злоумышленник, способный вмешиваться в канал управления или выдавать себя за него, может установить несанкционированную прошивку, что приведет к выполнению кода с высокими привилегиями. Оценка CVSS v3 для этой уязвимости составляет 9.8, что указывает на критический уровень серьезности.

Компания XCharge развернула обновление прошивки для всех затронутых зарядных устройств по состоянию на 22 мая 2026 года. Пользователям рекомендуется убедиться, что их устройства обновлены, и при необходимости обратиться в службу поддержки XCharge. Затронутый продукт широко используется в секторе транспортных систем в нескольких странах.

Для аналитиков SOC: приоритетно мониторить интерфейсы управления зарядных устройств XCharge C6 на предмет несанкционированного доступа или аномальных запросов на обновление прошивки. Командам DevSecOps следует обеспечить сегментацию сети и немедленно применить патч от вендора, так как отсутствие проверок целостности делает эти устройства привлекательной целью для атак на цепочку поставок.
