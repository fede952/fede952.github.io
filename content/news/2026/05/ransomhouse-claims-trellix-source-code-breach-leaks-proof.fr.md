Le groupe menaçant RansomHouse revendique la responsabilité de la violation du dépôt de code source de Trellix, divulguant des images comme preuve de l'intrusion.

{{< cyber-report severity="High" source="BleepingComputer" target="dépôt de code source de Trellix" >}}

Le groupe menaçant RansomHouse a revendiqué la responsabilité de la violation du dépôt de code source de Trellix, divulguée la semaine dernière. Comme preuve de l'intrusion, le groupe a divulgué un petit ensemble d'images, indiquant qu'ils avaient accès à du code sensible.

{{< ad-banner >}}

L'attaque met en évidence les risques persistants pour les chaînes d'approvisionnement logicielles, où les dépôts de code source sont des cibles privilégiées pour les acteurs malveillants. Bien que l'ampleur complète de la violation ne soit pas encore connue, l'incident souligne la nécessité de contrôles d'accès robustes et d'une surveillance des environnements de développement.

Trellix n'a pas encore publié de déclaration détaillée sur l'impact de la violation. Il est conseillé aux équipes de sécurité de revoir leurs propres pratiques de sécurité des dépôts de code et de s'assurer que l'authentification multifacteur et l'accès au moindre privilège sont appliqués.

{{< netrunner-insight >}}

Pour les analystes SOC, surveillez les schémas d'accès inhabituels aux dépôts de code source et envisagez de déployer des honeytokens. Les équipes DevSecOps doivent appliquer des règles strictes de protection des branches et auditer toutes les intégrations tierces avec les plateformes d'hébergement de code.
