Критическая уязвимость в конфигурациях почтового агента Exim может позволить неаутентифицированным злоумышленникам удаленно выполнять произвольный код. Немедленно установите исправление.

{{< cyber-report severity="Critical" source="BleepingComputer" target="почтовый агент Exim" >}}

Обнаружена критическая уязвимость в почтовом агенте Exim с открытым исходным кодом, затрагивающая определенные конфигурации. Эта ошибка может позволить неаутентифицированному удаленному злоумышленнику выполнять произвольный код на уязвимых системах.

{{< ad-banner >}}

Exim широко используется в качестве почтового сервера в Unix-подобных системах, что делает эту уязвимость особенно опасной для организаций, полагающихся на него для доставки электронной почты. Точные технические детали эксплойта не были полностью раскрыты, но уровень серьезности указывает на необходимость немедленного применения исправлений.

Администраторам следует проверить свои конфигурации Exim и применить все доступные обновления от проекта Exim. До установки исправлений рассмотрите возможность внедрения средств контроля доступа на уровне сети, чтобы ограничить доступ к уязвимому сервису.

{{< netrunner-insight >}}

Это критический вектор удаленного выполнения кода в широко распространенном MTA. Аналитикам SOC следует в первую очередь сканировать экземпляры Exim и проверять усиление конфигурации. Команды DevSecOps должны ускорить установку исправлений и рассмотреть правила WAF для блокировки попыток эксплуатации до применения обновлений.
