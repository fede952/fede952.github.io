Trellix confirma acesso não autorizado a uma parte de seu repositório de código fonte, trabalhando com especialistas forenses e autoridades policiais para investigar a violação.

{{< cyber-report severity="High" source="The Hacker News" target="repositório de código fonte da Trellix" >}}

A empresa de segurança cibernética Trellix divulgou um incidente de segurança envolvendo acesso não autorizado a uma parte de seu repositório de código fonte. A empresa afirmou que 'identificou recentemente' o comprometimento e imediatamente contratou especialistas forenses líderes para investigar e remediar o problema.

{{< ad-banner >}}

A Trellix também notificou as autoridades policiais sobre a violação, embora ainda não tenha divulgado a extensão do código exposto ou se algum dado de cliente foi afetado. O anúncio não incluiu detalhes técnicos específicos sobre o vetor de ataque ou o período da invasão.

Como um grande player na indústria de segurança cibernética, a violação do código fonte da Trellix levanta preocupações sobre potencial roubo de propriedade intelectual e a possibilidade de agentes de ameaças analisarem o código em busca de vulnerabilidades. A empresa não indicou qualquer impacto em seus produtos ou serviços até o momento.

{{< netrunner-insight >}}

Para analistas de SOC, este incidente ressalta a importância de monitorar logs de acesso a repositórios e implementar controles de acesso rigorosos para código fonte. Equipes de DevSecOps devem revisar suas próprias permissões de repositório de código e considerar habilitar regras de proteção de branch e registro de auditoria para detectar acesso não autorizado precocemente.
