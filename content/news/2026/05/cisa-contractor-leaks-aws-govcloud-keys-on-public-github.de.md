Abgeordnete fordern Aufklärung, nachdem ein CISA-Auftragnehmer absichtlich AWS GovCloud-Schlüssel und andere Behördengeheimnisse auf einem öffentlichen GitHub-Konto veröffentlicht hat, während die CISA versucht, den Vorfall einzudämmen.

{{< cyber-report severity="High" source="Krebs on Security" target="CISA AWS GovCloud-Umgebung" >}}

Abgeordnete beider Kammern des Kongresses fordern Aufklärung von der US-amerikanischen Cybersecurity & Infrastructure Security Agency (CISA), nachdem KrebsOnSecurity berichtete, dass ein CISA-Auftragnehmer absichtlich AWS GovCloud-Schlüssel und eine große Menge anderer Behördengeheimnisse auf einem öffentlichen GitHub-Konto veröffentlicht hat. Der Vorfall, bei dem sensible Anmeldedaten und Daten offengelegt wurden, hat dringende Anfragen von Gesetzgebern ausgelöst, die sich um die Sicherheitspraktiken der Behörde sorgen.

{{< ad-banner >}}

Die CISA hat derzeit Schwierigkeiten, den Vorfall einzudämmen und die durchgesickerten Anmeldedaten zu widerrufen. Der Vorfall unterstreicht die Risiken, die mit dem Zugriff von Auftragnehmern auf sensible Systeme verbunden sind, sowie die Herausforderungen bei der Sicherung von Cloud-Umgebungen, insbesondere solcher, die von Regierungsbehörden genutzt werden. Die Behörde hat weder das volle Ausmaß der offengelegten Daten noch die Identität des beteiligten Auftragnehmers bekannt gegeben.

{{< netrunner-insight >}}

Dieser Vorfall unterstreicht die dringende Notwendigkeit strenger Zugriffskontrollen und kontinuierlicher Überwachung der Aktivitäten von Auftragnehmern in Cloud-Umgebungen. SOC-Analysten sollten die Überprüfung von GitHub-Repositorys auf durchgesickerte Anmeldedaten priorisieren und automatisierte Tools zur Erkennung von Geheimnissen implementieren. DevSecOps-Teams müssen das Prinzip der geringsten Privilegien durchsetzen und sicherstellen, dass alle Cloud-Schlüssel sofort rotiert werden, sobald ein Verdacht auf Offenlegung besteht.
