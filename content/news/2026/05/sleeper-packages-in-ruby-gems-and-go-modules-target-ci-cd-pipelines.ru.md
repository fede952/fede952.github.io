Злоумышленники используют спящие пакеты для доставки вредоносных нагрузок, кражи учетных данных, вмешательства в GitHub Actions и установки SSH-персистентности в атаках на цепочки поставок программного обеспечения.

{{< cyber-report severity="High" source="The Hacker News" target="CI/CD конвейеры и цепочки поставок программного обеспечения" >}}

Обнаружена новая кампания атак на цепочки поставок программного обеспечения, в которой используются спящие пакеты в качестве канала для последующей отправки вредоносных нагрузок, обеспечивающих кражу учетных данных, вмешательство в GitHub Actions и SSH-персистентность. Активность приписывается учетной записи GitHub "BufferZoneCorp", которая опубликовала набор репозиториев, связанных с вредоносными Ruby gems и Go modules.

{{< ad-banner >}}

Атака использует изначально безобидные пакеты, которые позже получают вредоносные обновления — метод, известный как "спящие" или "троянизированные" пакеты. После установки в средах CI/CD вредоносные нагрузки крадут учетные данные, изменяют рабочие процессы GitHub Actions и устанавливают постоянный SSH-доступ, что представляет значительную угрозу для конвейеров разработки.

Организациям, использующим Ruby gems или Go modules из ненадежных источников, следует провести аудит своих зависимостей и отслеживать подозрительную активность в репозиториях. Кампания подчеркивает растущую сложность атак на цепочки поставок, нацеленных на инфраструктуру разработчиков.

{{< netrunner-insight >}}

Эта кампания подчеркивает необходимость строгой фиксации зависимостей и проверки целостности в CI/CD конвейерах. Аналитикам SOC следует отслеживать аномальные изменения GitHub Actions и добавления SSH-ключей, а инженерам DevSecOps — внедрять доступ с минимальными привилегиями и рассмотреть использование эфемерных сред сборки для ограничения радиуса поражения.
