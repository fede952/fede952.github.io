CISA предупреждает о CVE-2025-40948, затрагивающей несколько устройств Ruggedcom ROX. Аутентифицированный удаленный злоумышленник может читать произвольные файлы с привилегиями root. Обновитесь до версии 2.17.1 или новее.

{{< cyber-report severity="Medium" source="CISA" target="Устройства Siemens Ruggedcom ROX" cve="CVE-2025-40948" cvss="6.8" >}}

Устройства серии Siemens Ruggedcom ROX подвержены уязвимости неправильного контроля доступа (CVE-2025-40948), которая позволяет аутентифицированному удаленному злоумышленнику читать произвольные файлы с привилегиями root из базовой операционной системы. Проблема вызвана некорректной проверкой входных данных в интерфейсе JSON-RPC веб-сервера, что позволяет внедрять аргументы.

{{< ad-banner >}}

Следующие продукты уязвимы: RUGGEDCOM ROX MX5000, MX5000RE, RX1400, RX1500, RX1501, RX1510, RX1511, RX1512, RX1524, RX1536 и RX5000, все версии до 2.17.1. Siemens выпустила обновления для устранения проблемы и рекомендует немедленно установить исправления.

С оценкой CVSS v3 6.8 эта уязвимость имеет средний уровень серьезности. Вектор атаки сетевой, требует низких привилегий и не требует взаимодействия с пользователем. Учитывая секторы критической инфраструктуры (например, критическое производство), где используются эти устройства, эксплуатация может привести к значительному раскрытию информации.

{{< netrunner-insight >}}

Для аналитиков SOC: приоритетно установите исправления на устройства Ruggedcom ROX в вашей среде, особенно на те, которые доступны из ненадежных сетей. Аутентифицированный характер эксплойта снижает непосредственный риск, но не устраняет его — злоумышленники, скомпрометировавшие учетную запись с низкими привилегиями, могут получить полный доступ к корневым файлам. Командам DevSecOps следует пересмотреть усиление конечной точки JSON-RPC и рассмотреть сегментацию сети для ограничения воздействия.
