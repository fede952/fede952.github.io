Множественные уязвимости в ABB B&R Automation Runtime до версии 6.4 могут позволить злоумышленникам захватывать сессии или выполнять код. В рекомендации CISA ICSA-26-141-04 описаны исправления.

{{< cyber-report severity="Medium" source="CISA" target="ABB B&R Automation Runtime" cve="CVE-2025-3449" cvss="6.1" >}}

CISA выпустила рекомендацию ICSA-26-141-04, в которой описаны множественные уязвимости в ABB B&R Automation Runtime — программной платформе, используемой в промышленной автоматизации. Недостатки, выявленные в ходе внутреннего анализа безопасности B&R, затрагивают версии до 6.4 и включают CVE-2025-3449 (предсказуемые идентификаторы сессий), CVE-2025-3448 (межсайтовый скриптинг) и CVE-2025-11498 (некорректная нейтрализация элементов формул в CSV-файлах). Неаутентифицированный злоумышленник может использовать их для захвата удаленных сессий или выполнения кода в контексте браузера пользователя.

{{< ad-banner >}}

Наиболее критичная уязвимость, CVE-2025-3449, находится в компоненте System Diagnostic Manager (SDM) и имеет оценку CVSS v3 6.1. Она позволяет неаутентифицированному сетевому злоумышленнику перехватывать уже установленные сессии из-за генерации предсказуемых чисел или идентификаторов. По умолчанию SDM отключен в Automation Runtime 6, что снижает риск, но организациям следует проверить, что он остается выключенным, если явно не требуется.

ABB выпустила версию Automation Runtime 6.4 для устранения этих проблем. Учитывая развертывание продукта в энергетическом секторе по всему миру, CISA настоятельно рекомендует операторам как можно скорее установить обновление. В рекомендации отмечается, что успешная эксплуатация может привести к удаленному выполнению кода или захвату сессии, что представляет значительный риск для сред промышленной автоматизации.

{{< netrunner-insight >}}

Для аналитиков SOC: приоритетно установите исправления на экземпляры Automation Runtime, особенно с включенным SDM. Уязвимость предсказуемого идентификатора сессии (CVE-2025-3449) тривиально эксплуатируется по сети. Командам DevSecOps следует убедиться, что SDM отключен в производственной среде, и проверить, что ни один из открытых экземпляров не доступен из ненадежных сетей. Отслеживайте подозрительную активность сессий как сигнал обнаружения.
