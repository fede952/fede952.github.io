Trellixは、ソースコードリポジトリの一部への不正アクセスを確認し、フォレンジック専門家や法執行機関と協力して侵害の調査を進めている。

{{< cyber-report severity="High" source="The Hacker News" target="Trellixソースコードリポジトリ" >}}

サイバーセキュリティ企業Trellixは、ソースコードリポジトリの一部への不正アクセスを伴うセキュリティインシデントを開示した。同社は「最近特定した」と述べ、直ちに大手フォレンジック専門家を招き、問題の調査と是正に取り組んでいる。

{{< ad-banner >}}

Trellixはまた、法執行機関に侵害を通知したが、公開されたコードの範囲や顧客データへの影響についてはまだ明らかにしていない。発表には、攻撃ベクトルや侵入の期間に関する具体的な技術的詳細は含まれていなかった。

サイバーセキュリティ業界の主要プレーヤーとして、Trellixのソースコード侵害は、潜在的な知的財産の盗難や、脅威アクターがコードを分析して脆弱性を探る可能性について懸念を引き起こしている。同社は現時点で、製品やサービスへの影響はないとしている。

SOCアナリストにとって、このインシデントはリポジトリのアクセスログを監視し、ソースコードに厳格なアクセス制御を実装することの重要性を強調している。DevSecOpsチームは、自社のコードリポジトリの権限を見直し、ブランチ保護ルールや監査ログを有効にして不正アクセスを早期に検出することを検討すべきである。
