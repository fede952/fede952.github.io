Microsoft behebt 167 Sicherheitslücken, darunter einen SharePoint Zero-Day und eine öffentlich bekannt gewordene Windows Defender Schwachstelle (BlueHammer). Google Chrome und Adobe Reader patchen ebenfalls aktiv ausgenutzte Fehler.

{{< cyber-report severity="Critical" source="Krebs on Security" target="Microsoft Windows, SharePoint, Windows Defender, Chrome, Adobe Reader" >}}

Der Microsoft Patch Tuesday vom April 2026 behebt satte 167 Sicherheitslücken in Windows und zugehöriger Software. Zu den kritischsten gehört eine SharePoint Server Zero-Day-Schwachstelle, die Remote-Codeausführung ermöglichen könnte, obwohl im Bericht keine CVE-Kennung angegeben wurde. Zudem wurde eine öffentlich bekannt gewordene Schwachstelle in Windows Defender namens 'BlueHammer' behoben.

{{< ad-banner >}}

Separat hat Google Chrome seinen vierten Zero-Day des Jahres 2026 gepatcht, was den Trend häufiger Browser-Updates fortsetzt. Adobe Reader erhielt ebenfalls ein Notfall-Update, um eine aktiv ausgenutzte Schwachstelle zu beheben, die zu Remote-Codeausführung führen kann. Organisationen sollten diese Updates aufgrund der aktiven Ausnutzung priorisieren.

Die schiere Menge an Patches in diesem Monat unterstreicht die Bedeutung robuster Patch-Management-Prozesse. Sicherheitsteams sollten sich auf den SharePoint Zero-Day und das Windows Defender Problem als sofortige Prioritäten konzentrieren und gleichzeitig sicherstellen, dass Chrome und Adobe Reader im gesamten Unternehmen aktualisiert werden.

Für SOC-Analysten: Priorisieren Sie den SharePoint Zero-Day und die BlueHammer Windows Defender Schwachstelle für sofortige Patches, da sie entweder aktiv ausgenutzt oder öffentlich bekannt sind. DevSecOps-Teams sollten diese Updates in ihre CI/CD-Pipelines integrieren und überprüfen, dass Endpunktschutztools durch das Defender-Update nicht gestört werden. Die Chrome- und Adobe Reader-Patches erfordern ebenfalls dringende Aufmerksamkeit aufgrund ihres aktiven Ausnutzungsstatus.
