Trellix confirme un accès non autorisé à une partie de son référentiel de code source, travaillant avec des experts médico-légaux et les forces de l'ordre pour enquêter sur la violation.

{{< cyber-report severity="High" source="The Hacker News" target="Référentiel de code source de Trellix" >}}

L'entreprise de cybersécurité Trellix a divulgué un incident de sécurité impliquant un accès non autorisé à une partie de son référentiel de code source. La société a déclaré avoir « récemment identifié » la compromission et avoir immédiatement engagé des experts médico-légaux de premier plan pour enquêter et remédier au problème.

{{< ad-banner >}}

Trellix a également informé les forces de l'ordre de la violation, bien qu'elle n'ait pas encore divulgué l'étendue du code exposé ni si des données clients ont été affectées. L'annonce n'incluait pas de détails techniques spécifiques sur le vecteur d'attaque ou la période de l'intrusion.

En tant qu'acteur majeur de l'industrie de la cybersécurité, la violation du code source de Trellix soulève des inquiétudes concernant un éventuel vol de propriété intellectuelle et la possibilité que des acteurs malveillants analysent le code à la recherche de vulnérabilités. La société n'a indiqué aucun impact sur ses produits ou services à ce stade.

{{< netrunner-insight >}}

Pour les analystes SOC, cet incident souligne l'importance de surveiller les journaux d'accès aux référentiels et de mettre en œuvre des contrôles d'accès stricts pour le code source. Les équipes DevSecOps devraient examiner leurs propres autorisations de référentiel de code et envisager d'activer les règles de protection des branches et la journalisation d'audit pour détecter rapidement les accès non autorisés.
