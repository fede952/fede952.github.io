Lawmakers demand answers after a CISA contractor intentionally published AWS GovCloud keys and agency secrets on a public GitHub account, as CISA struggles to contain the breach.

{{< cyber-report severity="High" source="Krebs on Security" target="CISA AWS GovCloud environment" >}}

Lawmakers in both houses of Congress are demanding answers from the U.S. Cybersecurity & Infrastructure Security Agency (CISA) after KrebsOnSecurity reported that a CISA contractor intentionally published AWS GovCloud keys and a vast trove of other agency secrets on a public GitHub account. The breach, which exposed sensitive credentials and data, has prompted urgent inquiries from legislators concerned about the agency's security practices.

{{< ad-banner >}}

CISA is currently struggling to contain the breach and invalidate the leaked credentials. The incident highlights the risks associated with contractor access to sensitive systems and the challenges of securing cloud environments, particularly those used by government agencies. The agency has not yet disclosed the full extent of the data exposed or the identity of the contractor involved.

{{< netrunner-insight >}}

This incident underscores the critical need for strict access controls and continuous monitoring of contractor activities in cloud environments. SOC analysts should prioritize auditing GitHub repositories for leaked credentials and implementing automated secret scanning tools. DevSecOps teams must enforce least-privilege access and ensure that all cloud keys are rotated immediately upon any suspicion of exposure.
