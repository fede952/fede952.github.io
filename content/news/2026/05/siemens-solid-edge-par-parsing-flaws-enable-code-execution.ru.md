Две уязвимости парсинга файлов в Siemens Solid Edge SE2026 позволяют злоумышленникам выполнить произвольный код через специально созданные PAR-файлы. Обновитесь до V226.0 Update 5.

{{< cyber-report severity="High" source="CISA" target="Siemens Solid Edge SE2026" cve="CVE-2026-44411" cvss="7.8" >}}

Siemens Solid Edge SE2026 до Update 5 подвержен двум уязвимостям парсинга файлов, которые могут быть вызваны при чтении приложением специально созданных PAR-файлов. Недостатки включают доступ к неинициализированному указателю (CVE-2026-44411) и переполнение буфера в стеке (CVE-2026-44412), оба из которых могут позволить злоумышленнику вызвать сбой приложения или выполнить произвольный код в контексте текущего процесса.

{{< ad-banner >}}

Уязвимости имеют базовую оценку CVSS v3.1 7.8 (High) с вектором AV:L/AC:L/PR:N/UI:R/S:U/C:H/I:H/A:H, что указывает на локальный доступ, низкую сложность, отсутствие необходимых привилегий, необходимость взаимодействия с пользователем и высокое влияние на конфиденциальность, целостность и доступность. Siemens выпустил версию V226.0 Update 5 для устранения этих проблем и рекомендует пользователям немедленно обновиться.

Учитывая критическое развертывание в производственном секторе по всему миру, организациям, использующим Solid Edge, следует уделить первоочередное внимание установке исправлений. Уязвимости требуют взаимодействия с пользователем (открытие вредоносного PAR-файла), поэтому также рекомендуется обучение пользователей в качестве компенсирующего контроля.

{{< netrunner-insight >}}

Для аналитиков SOC: отслеживайте необычную обработку PAR-файлов или сбои в процессах Solid Edge. Инженеры DevSecOps должны внедрить белые списки приложений и ограничить типы файлов для уменьшения поверхности атаки. Поскольку это локальные уязвимости, зависящие от взаимодействия с пользователем, ключевыми мерами смягчения являются фишинговые симуляции и правила обнаружения конечных точек для подозрительных открытий файлов.
