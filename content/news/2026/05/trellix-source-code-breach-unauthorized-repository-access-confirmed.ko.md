Trellix는 소스 코드 저장소의 일부에 대한 승인되지 않은 접근을 확인하고, 포렌식 전문가 및 법 집행 기관과 협력하여 침해를 조사하고 있습니다.

{{< cyber-report severity="High" source="The Hacker News" target="Trellix 소스 코드 저장소" >}}

사이버 보안 기업 Trellix가 소스 코드 저장소의 일부에 대한 승인되지 않은 접근을 포함한 보안 사고를 공개했습니다. 회사는 '최근에' 해당 침해를 식별하고 즉시 주요 포렌식 전문가를 고용하여 문제를 조사하고 해결했다고 밝혔습니다.

{{< ad-banner >}}

Trellix는 또한 법 집행 기관에 침해 사실을 통보했지만, 노출된 코드의 범위나 고객 데이터가 영향을 받았는지 여부는 아직 공개하지 않았습니다. 이 발표에는 공격 벡터나 침해 시간대에 대한 구체적인 기술적 세부 정보는 포함되지 않았습니다.

사이버 보안 업계의 주요 기업인 Trellix의 소스 코드 침해는 잠재적인 지적 재산 도난과 위협 행위자가 코드를 분석하여 취약점을 찾을 가능성에 대한 우려를 제기합니다. 회사는 현재까지 제품이나 서비스에 미치는 영향은 없다고 밝혔습니다.

SOC 분석가에게 이 사건은 저장소 접근 로그를 모니터링하고 소스 코드에 대한 엄격한 접근 제어를 구현하는 것의 중요성을 강조합니다. DevSecOps 팀은 자체 코드 저장소 권한을 검토하고, 조기 침해 탐지를 위해 브랜치 보호 규칙 및 감사 로깅을 활성화하는 것을 고려해야 합니다.
