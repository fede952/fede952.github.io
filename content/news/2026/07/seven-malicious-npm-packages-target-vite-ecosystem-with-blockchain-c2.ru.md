Checkmarx раскрывает кампанию ViteVenom, использующую инфраструктуру C2 на основе блокчейна для доставки RAT через семь вредоносных пакетов npm, нацеленных на инструментарий фронтенда Vite.

{{< cyber-report severity="High" source="The Hacker News" target="Экосистема инструментария фронтенда Vite" >}}

Исследователи в области кибербезопасности из Checkmarx выявили группу из семи вредоносных пакетов npm, нацеленных на экосистему инструментария фронтенда Vite в рамках атаки на цепочку поставок программного обеспечения. Кампания, получившая кодовое название ViteVenom, представляет собой расширение ранее наблюдавшейся операции ChainVeil, которая использовала беспрецедентную четырехуровневую инфраструктуру управления и контроля (C2) на основе блокчейна, охватывающую сеть Tron.

{{< ad-banner >}}

Вредоносные пакеты предназначены для доставки трояна удаленного доступа (RAT) на скомпрометированные системы, позволяя злоумышленникам похищать данные и поддерживать постоянный доступ. Использование блокчейна для связи C2 усложняет обнаружение и нейтрализацию, так как инфраструктура децентрализована и устойчива к традиционным методам перехвата.

Организации, использующие Vite в своих конвейерах разработки, должны немедленно проверить свои зависимости на наличие выявленных вредоносных пакетов и внедрить строгие проверки целостности пакетов. Этот инцидент подчеркивает растущую сложность атак на цепочку поставок программного обеспечения, когда злоумышленники используют легитимные инструменты разработки и децентрализованные сети для уклонения от обнаружения.

{{< netrunner-insight >}}

Для аналитиков SOC мониторинг исходящих соединений к узлам блокчейна и необычных DNS-запросов может помочь обнаружить эту технику C2. Команды DevSecOps должны обеспечить подпись пакетов и использовать инструменты сканирования зависимостей для блокировки известных вредоносных пакетов до их попадания в конвейер сборки.
