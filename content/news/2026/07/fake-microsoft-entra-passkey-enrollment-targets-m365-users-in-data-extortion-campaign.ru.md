Угроза O-UNC-066 использует голосовой фишинг, чтобы обманом заставить пользователей зарегистрировать поддельный ключ доступа Entra, с целью компрометации учетных записей Microsoft 365 для вымогательства данных.

{{< cyber-report severity="High" source="The Hacker News" target="Пользователи Microsoft 365" >}}

Угроза, отслеживаемая Okta как O-UNC-066, была замечена в проведении атак с использованием голосового фишинга, нацеленных на пользователей Microsoft 365 в различных секторах. Злоумышленники выдают себя за законные запросы безопасности, чтобы обманом заставить жертв зарегистрировать поддельный ключ доступа Entra, предоставляя тем самым несанкционированный доступ к их учетным записям.

{{< ad-banner >}}

Кампания использует фишинговый набор, управляемый через панель, специально разработанный для перехвата процесса регистрации ключа доступа. После получения доступа злоумышленники стремятся осуществить вымогательство данных, похищая конфиденциальную информацию и требуя выкуп. Атаки подчеркивают растущую тенденцию использования голосовых каналов для обхода традиционных защит на основе фишинга по электронной почте.

Организациям рекомендуется внедрять многофакторную аутентификацию (MFA) с аппаратными ключами безопасности и обучать пользователей проверять любые незапрошенные запросы безопасности через альтернативные каналы связи. Мониторинг подозрительных действий по регистрации ключей доступа может помочь обнаружить такие атаки на ранней стадии.

{{< netrunner-insight >}}

Эта атака подчеркивает важность отношения к голосовым запросам безопасности с той же степенью скептицизма, что и к фишинговым электронным письмам. Аналитикам SOC следует отслеживать необычные попытки регистрации ключей доступа и убедиться, что процессы регистрации MFA требуют проверки вне полосы. Командам DevSecOps следует рассмотреть возможность внедрения политик условного доступа, ограничивающих регистрацию ключей доступа доверенными устройствами и местоположениями.
