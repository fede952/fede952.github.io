A new data-extortion group called Helix uses vishing, device code phishing, and MFA abuse to steal data from SharePoint environments.

{{< cyber-report severity="High" source="BleepingComputer" target="SharePoint environments" >}}

A new data-extortion group named Helix has emerged, employing identity-focused tactics to compromise SharePoint environments. The group leverages voice phishing (vishing), device code phishing, and multi-factor authentication (MFA) abuse to gain initial access and steal sensitive data.

{{< ad-banner >}}

Helix's approach highlights a shift toward social engineering and credential theft rather than exploiting technical vulnerabilities. By targeting identity systems, the group can bypass traditional security controls and move laterally within victim networks.

Organizations using SharePoint should review their MFA policies and educate users about vishing and device code phishing. Monitoring for unusual authentication patterns and enforcing conditional access policies can help mitigate these attacks.

{{< netrunner-insight >}}

For SOC analysts, this group underscores the need to monitor for anomalous device code authentication requests and vishing attempts. DevSecOps teams should enforce phishing-resistant MFA and implement strict conditional access policies for cloud applications like SharePoint.
