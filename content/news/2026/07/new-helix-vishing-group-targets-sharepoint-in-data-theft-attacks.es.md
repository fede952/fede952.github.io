Un nuevo grupo de extorsión de datos llamado Helix utiliza vishing, phishing de código de dispositivo y abuso de MFA para robar datos de entornos de SharePoint.

{{< cyber-report severity="High" source="BleepingComputer" target="entornos de SharePoint" >}}

Ha surgido un nuevo grupo de extorsión de datos llamado Helix, que emplea tácticas centradas en la identidad para comprometer entornos de SharePoint. El grupo aprovecha el vishing (phishing por voz), el phishing de código de dispositivo y el abuso de la autenticación multifactor (MFA) para obtener acceso inicial y robar datos sensibles.

{{< ad-banner >}}

El enfoque de Helix destaca un cambio hacia la ingeniería social y el robo de credenciales en lugar de explotar vulnerabilidades técnicas. Al atacar sistemas de identidad, el grupo puede eludir los controles de seguridad tradicionales y moverse lateralmente dentro de las redes de las víctimas.

Las organizaciones que utilizan SharePoint deben revisar sus políticas de MFA y educar a los usuarios sobre el vishing y el phishing de código de dispositivo. Monitorear patrones de autenticación inusuales y aplicar políticas de acceso condicional puede ayudar a mitigar estos ataques.

{{< netrunner-insight >}}

Para los analistas del SOC, este grupo subraya la necesidad de monitorear solicitudes de autenticación de código de dispositivo anómalas e intentos de vishing. Los equipos de DevSecOps deben implementar MFA resistente al phishing y aplicar políticas estrictas de acceso condicional para aplicaciones en la nube como SharePoint.
