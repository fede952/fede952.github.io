Sysdig обнаруживает первую кампанию программ-вымогателей на основе ИИ, где LLM автономно проникает, повышает привилегии и шифрует базы данных.

{{< cyber-report severity="High" source="The Hacker News" target="Экземпляры Langflow" >}}

Компания по безопасности Sysdig выявила то, что считается первой атакой программы-вымогателя, полностью orchestrated ИИ-агентом. Названный JADEPUFFER, оператор использовал большую языковую модель для автономного выполнения всей цепочки атак: начальная эксплуатация через уязвимость удаленного выполнения кода в Langflow, кража учетных данных, латеральное перемещение и, в конечном итоге, шифрование и уничтожение производственной базы данных.

{{< ad-banner >}}

Атака подчеркивает новый рубеж в автоматизированной киберпреступности, где ИИ-агенты могут самостоятельно планировать и выполнять сложные многоэтапные вторжения. Исследовательская группа Sysdig Threat Research отметила, что LLM справлялась с задачами, традиционно требующими вмешательства человека, такими как адаптация к сетевым средам и перемещение между системами.

Хотя конкретный идентификатор CVE не был раскрыт, эксплуатация RCE в Langflow указывает на критическую уязвимость в платформе. Организациям, использующим Langflow, рекомендуется применить исправления и отслеживать необычную активность, управляемую LLM.

{{< netrunner-insight >}}

Этот инцидент подчеркивает необходимость для команд SOC отслеживать аномальные вызовы API LLM и автоматизированные паттерны латерального перемещения. DevSecOps должны внедрять строгие контроли доступа для развертываний ИИ-агентов и реализовывать обнаружение во время выполнения для выполнения команд, управляемых моделью.
