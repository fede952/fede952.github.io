Un nouveau groupe d'extorsion de données appelé Helix utilise le vishing, le phishing par code d'appareil et l'abus d'authentification multifacteur pour voler des données dans les environnements SharePoint.

{{< cyber-report severity="High" source="BleepingComputer" target="environnements SharePoint" >}}

Un nouveau groupe d'extorsion de données nommé Helix a émergé, utilisant des tactiques axées sur l'identité pour compromettre les environnements SharePoint. Le groupe exploite le vishing (phishing vocal), le phishing par code d'appareil et l'abus d'authentification multifacteur (MFA) pour obtenir un accès initial et voler des données sensibles.

{{< ad-banner >}}

L'approche d'Helix met en évidence un glissement vers l'ingénierie sociale et le vol d'identifiants plutôt que l'exploitation de vulnérabilités techniques. En ciblant les systèmes d'identité, le groupe peut contourner les contrôles de sécurité traditionnels et se déplacer latéralement dans les réseaux des victimes.

Les organisations utilisant SharePoint devraient revoir leurs politiques MFA et sensibiliser les utilisateurs au vishing et au phishing par code d'appareil. La surveillance des schémas d'authentification inhabituels et l'application de politiques d'accès conditionnel peuvent aider à atténuer ces attaques.

{{< netrunner-insight >}}

Pour les analystes SOC, ce groupe souligne la nécessité de surveiller les demandes d'authentification par code d'appareil anormales et les tentatives de vishing. Les équipes DevSecOps devraient imposer une MFA résistante au phishing et mettre en œuvre des politiques d'accès conditionnel strictes pour les applications cloud comme SharePoint.
