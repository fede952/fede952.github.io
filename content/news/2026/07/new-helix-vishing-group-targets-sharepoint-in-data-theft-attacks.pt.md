Um novo grupo de extorsão de dados chamado Helix usa vishing, phishing de código de dispositivo e abuso de MFA para roubar dados de ambientes SharePoint.

{{< cyber-report severity="High" source="BleepingComputer" target="ambientes SharePoint" >}}

Um novo grupo de extorsão de dados chamado Helix surgiu, empregando táticas focadas em identidade para comprometer ambientes SharePoint. O grupo utiliza phishing por voz (vishing), phishing de código de dispositivo e abuso de autenticação multifator (MFA) para obter acesso inicial e roubar dados sensíveis.

{{< ad-banner >}}

A abordagem do Helix destaca uma mudança em direção à engenharia social e roubo de credenciais, em vez de explorar vulnerabilidades técnicas. Ao mirar sistemas de identidade, o grupo pode contornar controles de segurança tradicionais e se mover lateralmente dentro das redes das vítimas.

Organizações que usam SharePoint devem revisar suas políticas de MFA e educar os usuários sobre vishing e phishing de código de dispositivo. Monitorar padrões de autenticação incomuns e aplicar políticas de acesso condicional pode ajudar a mitigar esses ataques.

{{< netrunner-insight >}}

Para analistas de SOC, este grupo ressalta a necessidade de monitorar solicitações anômalas de autenticação de código de dispositivo e tentativas de vishing. Equipes de DevSecOps devem impor MFA resistente a phishing e implementar políticas rigorosas de acesso condicional para aplicativos em nuvem como o SharePoint.
