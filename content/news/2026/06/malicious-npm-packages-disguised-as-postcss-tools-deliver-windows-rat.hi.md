तीन दुर्भावनापूर्ण npm पैकेज, जो PostCSS टूल्स के रूप में प्रस्तुत होते हैं, एक Windows रिमोट एक्सेस ट्रोजन वितरित करते पाए गए हैं। शोधकर्ता npm पैकेज स्थापित करते समय सावधानी बरतने की सलाह देते हैं।

{{< cyber-report severity="High" source="The Hacker News" target="npm उपयोगकर्ता, Windows सिस्टम" >}}

साइबर सुरक्षा शोधकर्ताओं ने तीन दुर्भावनापूर्ण npm पैकेज—aes-decode-runner-pro, postcss-minify-selector, और postcss-minify-selector-parser—की पहचान की है, जो एक Windows-आधारित रिमोट एक्सेस ट्रोजन (RAT) वितरित करने के लिए डिज़ाइन किए गए हैं। ये पैकेज पिछले महीने एक npm उपयोगकर्ता द्वारा प्रकाशित किए गए थे और कुल 1,016 डाउनलोड जमा कर चुके हैं, जो एक मध्यम लेकिन चिंताजनक वितरण का संकेत देता है।

{{< ad-banner >}}

पैकेज एक लोकप्रिय CSS पोस्ट-प्रोसेसर, वैध PostCSS टूल्स के रूप में प्रस्तुत होते हैं, ताकि डेवलपर्स को उन्हें स्थापित करने के लिए धोखा दिया जा सके। एक बार स्थापित होने के बाद, दुर्भावनापूर्ण कोड एक पेलोड निष्पादित करता है जो संक्रमित Windows मशीन तक रिमोट एक्सेस स्थापित करता है, संभावित रूप से हमलावरों को डेटा चुराने, अतिरिक्त मैलवेयर स्थापित करने या नेटवर्क के भीतर आगे बढ़ने की अनुमति देता है।

यह घटना npm पारिस्थितिकी तंत्र में टाइपोस्क्वाटिंग और निर्भरता भ्रम के चल रहे खतरे को उजागर करती है। डेवलपर्स को सलाह दी जाती है कि वे पैकेज के नामों की सावधानीपूर्वक जाँच करें, स्थापना से पहले स्रोत कोड की समीक्षा करें, और ऐसे जोखिमों को कम करने के लिए पैकेज अखंडता सत्यापन उपकरणों का उपयोग करें।

SOC विश्लेषकों और DevSecOps इंजीनियरों के लिए, यह एक अनुस्मारक है कि सख्त पैकेज उत्पत्ति जाँच लागू करें और असामान्य npm पैकेज स्थापनाओं की निगरानी करें। ज्ञात दुर्भावनापूर्ण पैकेजों के लिए स्वचालित स्कैनिंग लागू करने और डेवलपर्स को आँख बंद करके पैकेज नामों पर भरोसा करने के जोखिमों के बारे में शिक्षित करने पर विचार करें। अपेक्षाकृत कम डाउनलोड संख्या बताती है कि यह अभियान प्रारंभिक चरण में हो सकता है, इसलिए समान पैकेजों के लिए सक्रिय रूप से खोज करना उचित है।
