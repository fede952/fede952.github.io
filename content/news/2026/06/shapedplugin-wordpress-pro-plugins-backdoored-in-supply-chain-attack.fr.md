Plusieurs plugins Pro WordPress de ShapedPlugin ont été compromis via une attaque de la chaîne d'approvisionnement, avec du code malveillant injecté dans les versions officielles.

{{< cyber-report severity="High" source="The Hacker News" target="Plugins Pro WordPress de ShapedPlugin" >}}

Plusieurs plugins WordPress de ShapedPlugin ont été compromis lors d'une attaque de la chaîne d'approvisionnement après que des acteurs malveillants inconnus ont réussi à altérer les canaux de distribution officiels et à injecter du code malveillant. Selon Wordfence, les attaquants ont compromis le pipeline de construction et de distribution du fournisseur, injectant du code malveillant dans les versions Pro des plugins distribuées via les canaux de mise à jour officiels sous licence.

{{< ad-banner >}}

Cette attaque met en évidence les risques associés aux écosystèmes de plugins tiers, où un seul fournisseur compromis peut impacter de nombreux sites web. Les utilisateurs des plugins Pro de ShapedPlugin sont invités à vérifier l'intégrité de leurs installations et à mettre à jour vers les dernières versions corrigées si disponibles.

Wordfence a publié une analyse détaillée du code malveillant, qui peut être utilisée pour détecter les installations compromises. Les organisations doivent examiner leurs environnements WordPress pour détecter tout signe d'accès non autorisé ou d'activité malveillante.

{{< netrunner-insight >}}

Cette attaque de la chaîne d'approvisionnement souligne le besoin crucial de contrôles de sécurité de la chaîne d'approvisionnement logicielle. Les analystes SOC doivent surveiller les comportements anormaux de mise à jour des plugins et envisager de mettre en œuvre des contrôles d'intégrité pour tout code tiers. Les équipes DevSecOps doivent renforcer la sécurité du pipeline et la signature de code pour éviter des compromissions similaires.
