Вредоносный npm-пакет codexui-android нацелен на разработчиков, похищая токены аутентификации OpenAI Codex; еженедельно загружается более 29 000 раз.

{{< cyber-report severity="High" source="The Hacker News" target="Разработчики OpenAI Codex" >}}

Исследователи в области кибербезопасности обнаружили вредоносную кампанию по атаке на цепочку поставок, нацеленную на разработчиков, использующих OpenAI Codex. Атака использует легитимно выглядящий npm-пакет с именем codexui-android, который рекламируется как удаленный веб-интерфейс для OpenAI Codex на GitHub и npm. Пакет привлек более 29 000 еженедельных загрузок, что указывает на значительный охват среди сообщества разработчиков.

{{< ad-banner >}}

Вредоносный пакет предназначен для кражи токенов аутентификации OpenAI Codex у ничего не подозревающих разработчиков. На момент отчета пакет остается доступным для загрузки, представляя постоянную угрозу. Разработчикам, установившим codexui-android, рекомендуется немедленно сменить свои токены и проверить системы на предмет несанкционированного доступа.

Этот инцидент подчеркивает постоянный риск атак на цепочку поставок в экосистеме открытого исходного кода. Использование легитимно звучащих имен пакетов и высокое количество загрузок может усыпить бдительность разработчиков. Организациям следует внедрить строгие процессы проверки пакетов и рассмотреть возможность использования инструментов, обнаруживающих аномальное поведение пакетов.

Для аналитиков SOC и инженеров DevSecOps эта атака подчеркивает необходимость мониторинга загрузок и поведения npm-пакетов. Внедрите обнаружение в реальном времени для неожиданной эксфильтрации токенов и применяйте принцип минимальных привилегий для токенов API. Регулярно проверяйте цепочку поставок программного обеспечения и рассмотрите возможность использования инструментов проверки целостности пакетов.
